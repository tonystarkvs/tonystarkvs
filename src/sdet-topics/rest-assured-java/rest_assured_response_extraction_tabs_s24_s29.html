Overview — Groovy GPath $ Syntax
🦊
What
GPath $ = root of the JSON document. $.key = top-level field. $[0] = first element of root array. Used inside body() assertions.
💡
Why
Most concise assertion style — filter, aggregate, navigate nested JSON all inline inside then().body() without extracting to Java.
⏰
When
Use for all structural assertions inside then().body() — single fields, nested objects, array elements, filters, aggregates.
📍
Where
Inside REST Assured's then().body(expression, matcher). Evaluated by Groovy's JsonSlurper engine on the response body.
⚙️
How
$ = root. $.key same as "key". $[0] = first array element. Closures: findAll{it.qty>1}, find{}, collect{}. Aggregates: sum(), min(), max().

Keyword Glossary
$
Root of JSON document in GPath. body("$", notNullValue()) = asserts body is not empty.
$.key
Root notation for top-level field. body("$.orderId", equalTo("X")) = same as body("orderId",...).
$[0]
First element of a root JSON array response. body("$[0].orderId") for array root responses.
dot notation
Navigate nested objects: "customer.address.city" — each dot descends one level.
items[0].tags[1]
Chained index access — first item's second tag. Arrays within arrays.
size()
Groovy List.size() — "items.size()" = count of items array elements.
findAll{}
Filter array: "items.findAll{it.qty>1}.productId" — returns List of productIds where qty>1.
find{}
Returns FIRST matching element only — singular result not a List.
sum() / min() / max()
Groovy aggregates on collected numeric fields: "items.price.sum()" = total of all prices.
it
Implicit Groovy closure parameter — the current element in findAll{}, find{}, collect{}.
Interview One-Liners
"$" = root of the JSON document — body("$", notNullValue()) asserts the response exists; body("$.orderId", equalTo("ORD-001")) is identical to body("orderId", equalTo("ORD-001")).
"$[0]" = first element of a root JSON array — only needed when the entire response IS an array. For object responses, just use "fieldName" or "$.fieldName".
"items.size()" = Groovy's size() on a JSON array field — counts elements without extracting the array to Java code.
findAll{it.qty > 1} = Groovy closure filtering array elements inline — no Java code, evaluated entirely by Groovy's engine inside body().
"items.price.sum()" = Groovy aggregate summing all price values across items — one expression replaces a Java stream mapToDouble().sum().
"items[0].tags[1]" = nested array indexing — first item's second tag. GPath supports chained [n] indexing for arrays within array elements.

Best Practices
✔ Use "$" prefix when the expression might be ambiguous — "$.status" is unambiguous about navigating from root.
✔ Prefer findAll{}.size() over extracting to Java and calling list.size() — keeps assertions inline and avoids extra steps.
✔ Use everyItem() for asserting all elements satisfy a condition: body("items.qty", everyItem(greaterThan(0))).
✔ GPath inside body() is for ASSERTIONS only — use jsonPath() or path() when you need to EXTRACT values to use in Java code.
✔ Combine path params, query params, and custom headers in a RequestSpecBuilder to avoid repeating them in every test method.
Interview Q&A
Q1: What does "$" mean in GPath and when do you need "$[0]"? +
"$" represents the ROOT of the JSON document. For object responses, "$.orderId" and "orderId" are identical — the $ is optional. You NEED "$[0]" specifically when the entire response body is a JSON array (not an object). For example, GET /users returns [{...},{...}], so body("$[0].name") accesses the first user's name. For object responses, just use dot notation without $.
Q2: What is the difference between find{} and findAll{} in GPath? +
findAll{condition} returns ALL elements where the closure returns true — result is a List. find{condition} returns only the FIRST element — result is a single object (not a list). Use findAll when asserting count or checking multiple matches. Use find when navigating into one specific element: body("items.find{it.productId=='P001'}.qty", equalTo(2)).
Q3: How do you access items[0].tags[1] — an array inside an array element? +
Use chained index notation: body("items[0].tags[1]", equalTo("api")). "items[0]" gets the first item object, ".tags" navigates to the tags field (which is an array), "[1]" gets the second element. For ALL tags across all items as a flat list: body("items.tags.flatten()", hasItem("java")) — flatten() merges nested arrays from all elements into one List.
Q4: How do you assert the sum of all prices with GPath? +
Use Groovy's sum(): body("items.price.sum()", equalTo(948.0f)). This evaluates as: collect all "price" values from items → [599.0, 349.0] → call .sum() on the Groovy List → 948.0. Similarly: body("items.price.min()", equalTo(349.0f)) and body("items.price.max()", equalTo(599.0f)) work on any collected numeric field array.
Q5: When would you prefer jsonPath() extraction over GPath body() assertions? +
Use jsonPath() when you need to EXTRACT a value to USE in subsequent Java code — e.g. store it for the next API call (API chaining), log it, or perform computation. GPath inside body() is assertion-only — it never returns values to Java. Also use jsonPath() for typed extraction: jp.getList("items.price", Float.class) returns a strongly-typed List<Float> rather than needing an Object cast.
Overview — response.path(expression)
🔍
What
response.path(expr) = quickest way to extract a single typed value from a stored Response object using a GPath expression. Returns Object — Java auto-casts.
💡
Why
More concise than creating a full JsonPath object for simple single-field extractions. Available on stored Response and in the fluent chain via extract().path().
⏰
When
When you have a Response stored and need one or a few values quickly. Also inside fluent chain: then().extract().path("orderId") terminates the chain.
📍
Where
On Response: res.path("customer.name"). In chain: then().extract().path("orderId"). Best pattern: store Response first with extract().response().
⚙️
How
REST Assured calls Groovy's JsonSlurper.parseText(body) internally and navigates the parsed tree using GPath. Returns the node as Object.
Keyword Glossary
response.path(expr)
Method on Response — evaluates GPath and returns matching value as Object. Auto-casts to String, int, List etc.
extract().path()
Fluent chain version — terminates then() and returns single value. No more assertions after this call.
extract().response()
Terminates chain, returns full Response object — allows both assertions AND multiple path() calls.
getStatusCode()
Method on Response returning the int HTTP status code (200, 201, etc.).
getTime()
Method on Response returning response duration in milliseconds — for SLA assertions.
getHeader(name)
Method on Response returning value of a specific response header by name.
asString()
Method on ResponseBody — returns entire response body as raw String text.
auto-cast
Java's implicit type narrowing — String name = res.path("name") works if JSON value is a string. Float needed for decimal numbers.
nested path
GPath dot notation in path(): "customer.address.city" evaluates 3 levels deep.
multi-value path
When expression selects from array without index: "items.productId" → returns List, not String.
Interview One-Liners
response.path("orderId") = fastest one-liner to extract a single value — GPath expression, returns Object that Java auto-casts to String/int/float/boolean.
extract().path("orderId") = terminates the fluent chain — cannot add any more .body() assertions after calling this in the same chain.
extract().response() = preferred when you need multiple values — store Response once, call path() as many times as needed with zero extra HTTP calls.
response.path("items[0].price") = path() fully supports GPath array indexing — same capability as body() assertions but returns the value instead of asserting it.
response.path("items.productId") = when path resolves to multiple values, it returns a List — always declare target variable as List<String> in this case.

Best Practices
✔ Always use extract().response() when you need more than one value — one HTTP call, unlimited path() calls on the stored Response object.
✔ When path() returns a List (no array index), declare the variable as List<String> — assigning to a String causes ClassCastException at runtime.
✔ Use res.getTime(), res.getStatusCode(), res.getHeader(name) for HTTP-level assertions alongside body path() extractions.
✔ All then().body() assertions must come BEFORE extract() — once extract() is called, the chain terminates.
✔ For decimal fields, declare as float not int — Jackson deserializes JSON decimals as Float/Double.
Interview Q&A
Q1: Difference between response.path() and jsonPath().getString()? +
response.path("field") returns Object — Java infers the type. jsonPath().getString("field") explicitly returns String and is type-safe. For numbers: path("qty") may require a cast; jsonPath().getInt("qty") always returns int. Both use GPath internally. Prefer path() for quick single-value extraction; prefer jsonPath() typed methods when type safety matters or when you need getList() for arrays.
Q2: Why can't you add body() assertions after extract().path() in the chain? +
extract() terminates the ValidatableResponse fluent chain and returns an ExtractableResponse. Once you call extract(), REST Assured finalizes the validation phase — you can't add more validators. Solution: put ALL body() assertions before extract(), then call extract().response() to get the full Response. Then use res.then().body() for additional assertions or res.path() for extractions.
Q3: What happens when path() expression matches multiple values? +
When GPath resolves to multiple elements — e.g. "items.productId" collects productId from every array element — path() returns a List<Object>. If you assign to a String variable, you get ClassCastException. Rule: if the path selects from an array without an index, it returns a List. Always check: does my expression have an array index? No → List. Yes → scalar.
Q4: How do you extract HTTP response metadata alongside body values? +
The Response object exposes: res.getStatusCode() for HTTP status, res.getTime() for response time in ms, res.getHeader("Content-Type") for a header, res.getHeaders() for all headers, res.getCookies() for cookie map, res.getBody().asString() for raw body text. These complement path() body extractions.
Q5: How do you access a 4-level deep field like customer.address.geo.lat? +
Use dot notation: res.path("customer.address.geo.lat"). REST Assured evaluates this as a GPath: root → customer object → address object → geo object → lat value. No depth limit — just chain dots. For decimals declare as float: float lat = res.path("customer.address.geo.lat");
Overview — jsonPath() Object
🗺️
What
response.jsonPath() = dedicated JsonPath object with typed getters: getString(), getInt(), getFloat(), getBoolean(), getList(path, Class), getMap(), getObject().
💡
Why
Eliminates unsafe implicit casting of path(). Typed methods prevent ClassCastException and make extraction intent explicit and IDE-friendly.
⏰
When
Use when extracting multiple fields — create JsonPath once, call typed getters for each field. Also use JsonPath.from(rawString) for string-based parsing.
📍
Where
JsonPath jp = response.jsonPath() — then jp.getString(), jp.getList("items.price", Float.class). Also JsonPath.from(rawString) for raw parsing.
⚙️
How
JsonPath wraps the body and surfaces same GPath engine through typed method signatures. getList(path, Class) avoids all casting.
Keyword Glossary
jsonPath()
Method on Response returning a JsonPath object pre-loaded with response body — ready for typed extractions.
JsonPath.from(String)
Static factory — creates JsonPath from raw JSON string. Useful for strings from files, DB, or other sources.
getString(path)
Type-safe String extraction — returns null if absent, never throws ClassCastException.
getInt(path)
Typed int extraction. getFloat(path) for decimals. getDouble(path) for high-precision.
getBoolean(path)
Extracts boolean — maps JSON true/false to Java boolean primitives.
getList(path, Class)
Typed list extraction — e.g. getList("items.price", Float.class) → List<Float>. No casting needed.
getMap(path)
Returns Map<String,Object> for a JSON object field — extracts a whole sub-object.
getObject(path, Class)
Deserializes JSON object at path into typed POJO — combines extraction and deserialization.
Groovy in getList()
Closures work in getList path: getList("items.findAll{it.qty>1}.productId", String.class).
parse once
JsonPath parses body once — all subsequent getter calls operate on the pre-parsed in-memory tree.
Interview One-Liners
jp.getString("customer.name") = type-safe String extraction — returns null if field absent, never throws ClassCastException unlike path() which returns Object.
jp.getList("items.price", Float.class) = typed list — the Class parameter tells JsonPath the element type, producing a clean List<Float> with no casting.
jp.getMap("customer.address") = extracts a whole JSON object as Map<String,Object> — key-based access to all fields without extracting each individually.
JsonPath.from(rawString) = static factory to parse ANY JSON string — no REST Assured Response needed, works on strings from files or DBs.
jp.getObject("fulfillment", FulfillmentModel.class) = deserializes a sub-object directly into a typed POJO — most powerful JsonPath method.

Best Practices
✔ Create the JsonPath object ONCE — all subsequent getter calls operate on the pre-parsed tree without re-parsing.
✔ Always use typed getList(path, String.class) — avoids ClassCastException when iterating the extracted list.
✔ Use getObject(path, Class) for sub-object POJO deserialization — cleaner than extracting each field individually.
✔ Groovy closures work in getList() paths: jp.getList("items.findAll{it.qty>1}.productId", String.class).
✔ Use getMap("field") when the sub-object structure is dynamic — safer than getObject() without a matching POJO.
Interview Q&A
Q1: Advantage of getList(path, Class) over getList(path)? +
getList(path) returns List<Object> — each element untyped, requires casting. getList(path, String.class) returns List<String> — no casting needed, compile-time type safety, IDE completion works. Always use the Class parameter form in production test code.
Q2: When to use JsonPath.from(String) vs response.jsonPath()? +
Use JsonPath.from(String) when: (1) you already extracted body as raw String with asString(), (2) parsing from a test resource file, (3) parsing JSON from a database result, (4) testing a JsonPath expression in isolation. response.jsonPath() is a convenience that calls JsonPath.from(response.asString()) internally.
Q3: How does getObject() differ from as(Class)? +
as(Class) deserializes the ENTIRE response body into the given class. getObject("fulfillment", FulfillmentModel.class) deserializes only the JSON subtree at the given path. Use getObject() when you only need one section of a large response and want to map it to a specific POJO without creating a full model for the entire response.
Q4: Can you use Groovy closures in getList() paths? +
Yes — the path argument in any JsonPath method is a GPath expression evaluated by Groovy. jp.getList("items.findAll{it.discount > 0}.productId", String.class) first filters items where discount > 0, then extracts productId from each matching element, returning List<String>. Groovy evaluation happens before the typed extraction — they compose seamlessly.
Q5: What is the difference between getMap() and getObject()? +
getMap(path) returns Map<String,Object> — dynamic key-value, no POJO needed. getObject(path, Class) deserializes into a typed POJO. Prefer getMap() when structure is dynamic or unknown. Prefer getObject() when you have a matching POJO and want compile-time type safety.
Overview — extract().as(Map.class)
🗂️
What
extract().as(Map.class) = deserializes the ENTIRE response JSON object into Map<String,Object>. Nested objects → LinkedHashMap. Arrays → ArrayList.
💡
Why
Complete response as navigable Java Map tree — useful for dynamic validation, iterating keys, or when you don't have a matching POJO class.
⏰
When
Dynamic/unknown schemas, exploratory tests, asserting all expected fields are present, or quick prototyping without creating POJOs.
📍
Where
In chain: then().extract().as(Map.class). Root must be a JSON object. For array responses use TypeRef<List<Map<String,Object>>>.
⚙️
How
REST Assured delegates to Jackson ObjectMapper. Produces LinkedHashMap for objects, ArrayList for arrays, Integer/Double for numbers.
Keyword Glossary
as(Map.class)
Jackson deserializes full JSON response → Map<String,Object>. Root must be a JSON object not array.
LinkedHashMap
Jackson's default Map for JSON objects — preserves key insertion order.
ArrayList
Jackson's default List for JSON arrays. Each element is a LinkedHashMap for object elements.
Number casting
JSON numbers become Integer or Double. Safe: ((Number) map.get("price")).doubleValue().
containsKey()
Map method — asserts field exists. Used for dynamic field-presence validation.
keySet()
Returns all keys as Set — useful for schema discovery and required-field validation.
forEach()
Java 8 Map method iterating all key-value pairs for logging or dynamic assertions.
nested Map cast
(Map) map.get("customer") — each nested JSON object requires a separate cast to access fields.
Stream on List
Java streams on the ArrayList from array fields — filter, map, count with typed operations.
unchecked cast
Casting Object to Map — compiler warns but safe when Jackson always returns Maps for JSON objects.
Interview One-Liners
as(Map.class) = Jackson maps JSON objects → LinkedHashMap, arrays → ArrayList, decimals → Double, integers → Integer.
Number casting: always use ((Number) map.get("price")).doubleValue() — Jackson may produce Integer or Double depending on value.
map.keySet() = validate all expected fields are present — iterate and assertContainsAll instead of individual body() assertions.
Nested access: (Map) map.get("customer") then (Map) customer.get("address") — each nested JSON object requires a cast.
Array access: (List) map.get("items") → cast each element to Map: ((Map) item).get("productId").

Q1: What Java types does Jackson produce when deserializing into Map.class? +
JSON object → LinkedHashMap (preserves insertion order). JSON array → ArrayList. JSON string → String. JSON integer (small) → Integer. JSON integer (large) → Long. JSON decimal → Double. JSON boolean → Boolean. JSON null → null. Always use (Number).intValue() or (Number).doubleValue() for numeric fields — never cast directly to int or double.
Q2: How do you navigate 4 levels deep using Map? +
Cast each level: Map customer = (Map) resp.get("customer"); Map address = (Map) customer.get("address"); Map geo = (Map) address.get("geo"); double lat = ((Number) geo.get("lat")).doubleValue(); Each (Map) cast is needed because get() returns Object. Q3: When to prefer Map over POJO? +
Prefer Map when: schema is dynamic/unknown at design time, quickly exploring a new API, need to iterate over all keys dynamically, or the response structure varies. Prefer POJO when: schema is fixed and well-known, multiple tests use same structure, nested access is frequent, or compile-time safety is important.

List.of("orderId","status","customer").forEach(f -> Assert.assertTrue(resp.containsKey(f), "Missing: " + f)); This catches missing fields without asserting specific values — useful for structural validation. It detects API regressions where the backend removes a response field.

Overview — JSONObject(rawString)
📦
What
new JSONObject(rawString) = org.json library's explicit JSON parser — wraps the response body string into JSONObject providing typed accessor methods with null-safe options.
💡
Why
Provides null-safe optString(), explicit getJSONObject() and getJSONArray() — cleaner than Map casting, no Groovy knowledge needed.
⏰
When
When project already uses org.json, or when you prefer explicit typed getters over GPath string expressions for every field access.
📍
Where
First: extract body with asString(). Then: new JSONObject(rawBody). Navigate nested objects via getJSONObject(key) and arrays via getJSONArray(key).
⚙️
How
org.json parser reads raw JSON string and builds an in-memory tree of JSONObject and JSONArray instances — no Jackson involved.
Keyword Glossary
JSONObject(String)
Constructor parsing raw JSON string. Throws JSONException on invalid JSON.
getString(key)
Returns String value. Throws JSONException if key missing or not a String.
optString(key)
Returns String or "" if key missing — NULL SAFE. optString(key, default) returns specified default.
getInt(key)
Returns int for numeric field. getDouble(key) for decimals.
getBoolean(key)
Returns boolean for true/false fields.
getJSONObject(key)
Returns nested JSONObject — used to navigate deeper levels.
getJSONArray(key)
Returns JSONArray for array fields. length() = count. getJSONObject(n) = nth element.
JSONArray(String)
Constructor for root JSON arrays. length() = element count.
has(key)
Returns boolean — true if key exists. Use before getString to avoid JSONException.
isNull(key)
Returns true if key exists but has JSON null value.
Interview One-Liners
new JSONObject(rawBody) = two-step: extract body as String with asString(), then wrap in JSONObject for typed key-based access.
optString("key","N/A") = null-safe — returns specified default if key absent, preventing JSONException for optional fields.
getJSONObject("address") = extracts nested JSON object as own JSONObject — enabling chained navigation: jo.getJSONObject("address").getJSONObject("geo").getDouble("lat").
getJSONArray("items").length() = counts elements — JSONArray.length() is org.json's equivalent of array.size().
has("key") = null-safe key existence check before accessing fields — use instead of try/catch around getString().

Best Practices
✔ Create one Gson instance per class — Gson is thread-safe after construction, reuse avoids repeated object creation.
✔ Use TypeToken<List<GsonItem>>(){}.getType() for array responses — never pass List.class directly to fromJson().
✔ Gson ignores unknown JSON fields by default — no @JsonIgnoreProperties needed unlike Jackson.
✔ Use @SerializedName when JSON key naming convention differs from Java naming (snake_case API → camelCase Java).
✔ Prefer Jackson's as(UserModel.class) over Gson for REST Assured projects — Jackson is auto-detected; Gson requires manual asString() + fromJson() steps.
Interview Q&A
Q1: Key difference between Gson and Jackson for deserialization? +
Jackson: requires getters/setters, needs @JsonIgnoreProperties for unknown fields, auto-detected by REST Assured so as(Class) works directly. Gson: no getters needed — field access directly, unknown fields ignored by default, requires manual asString() + fromJson() steps because REST Assured doesn't auto-detect Gson. Jackson is better integrated; Gson is simpler POJO design.
Q2: Why is TypeToken needed for Gson list deserialization? +
Java type erasure removes generic type info at runtime — List<GsonItem>.class doesn't exist, only List.class. Without TypeToken, Gson sees raw List and creates List<LinkedTreeMap>. TypeToken creates an anonymous subclass that captures List<GsonItem> in its class hierarchy — Gson can then read it via reflection at runtime to know the element type.
Q3: What does @SerializedName do in Gson? +
@SerializedName("json_key") tells Gson to map JSON key "json_key" to the annotated Java field, even if the field has a different name. Example: @SerializedName("order_id") String orderId — Gson reads JSON "order_id" and populates orderId field. Works for both deserialization (reading) and serialization (writing). Equivalent to Jackson's @JsonProperty.
Q4: How do you deserialize a 4-level nested JSON with Gson? +
Define nested POJO classes matching the JSON structure: GsonGeo{lat,lng}, GsonAddress{city,zip,GsonGeo geo}, GsonCustomer{name,GsonAddress address}, GsonOrder{GsonCustomer customer}. Gson deserializes recursively — when it encounters the "address" JSON object for a GsonAddress-typed field, it recursively deserializes that object into GsonAddress, which in turn deserializes "geo" into GsonGeo. One fromJson() call builds the entire object graph.
Q5: When to choose Gson vs Jackson vs jsonPath() for response extraction? +
jsonPath() for: quick extraction of 1–3 fields, GPath filtering/aggregation, no POJO class needed. Jackson as(Class) for: complete typed POJO graph, full REST Assured integration, enterprise frameworks. Gson fromJson() for: projects already using Gson (Retrofit/Android), simpler POJO design without getter boilerplate, team preference. Jackson + as(Class) = most powerful for complex nested responses. jsonPath() = most concise for specific field extraction.
