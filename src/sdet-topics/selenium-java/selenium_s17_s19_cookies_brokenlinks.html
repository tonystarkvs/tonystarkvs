Overview — Handling cookies in Selenium

A cookie is a small piece of key-value data stored in the browser by a website. Selenium provides the driver.manage() API to add, get, delete, and inspect cookies programmatically. Cookie manipulation is essential for bypassing login flows, injecting auth tokens, testing remember-me features, and resetting session state between tests.

What / Why / When / Where / How

WhatKey-value text data stored by browser per domain — controls sessions, preferences, tracking
WhySkip repetitive login steps, test session expiry, inject auth tokens, validate cookie presence
WhenBefore tests needing login state, after login to save session, between test methods for isolation
WhereIn @BeforeMethod for setup, @AfterMethod for cleanup, inside test for cookie assertions
Howdriver.manage().addCookie() / getCookies() / deleteCookie() after navigating to domain first

Keyword Glossary

Cookie
Java class in Selenium representing a browser cookie with name, value, domain, path, expiry
driver.manage()
Returns Options object — gateway to cookies, timeouts, window management
addCookie(cookie)
Injects a cookie into the current browser session for the loaded domain
getCookies()
Returns Set<Cookie> — all cookies for the current page domain
getCookieNamed(name)
Fetches a single Cookie object by its name string — returns null if not found
deleteCookie(cookie)
Removes a specific Cookie object from the browser session
deleteCookieNamed(name)
Removes cookie by name string — shortcut that avoids fetching object first
deleteAllCookies()
Removes ALL cookies for current domain — use to reset test state
Cookie.Builder
Fluent builder pattern to construct Cookie with domain, path, expiry, secure flag
SameSite
Cookie attribute controlling cross-site sending: Strict, Lax, None

Interview one-liners

You must navigate to the domain first before adding or reading cookies — Selenium enforces the same-origin policy.
driver.manage().addCookie() injects a cookie into the current browser session without any network request.
getCookieNamed() returns null if the cookie does not exist — always null-check before calling getValue().
deleteAllCookies() is called in @AfterMethod to ensure each test starts with a clean session state.
Cookie.Builder allows setting domain, path, expiry date, and secure/httpOnly flags programmatically.

Best Practices

  • ✓Always navigate to the target domain before calling any manage().addCookie() — WebDriver enforces same-origin policy; adding cookies for a domain not loaded throws InvalidCookieDomainException
  • ✓Always null-check the result of getCookieNamed() before calling .getValue() — it returns null if not found
  • ✓Call driver.navigate().refresh() after injecting cookies — some pages only read cookies on load, not mid-session
  • ✓Use deleteAllCookies() in @AfterMethod and @BeforeMethod to guarantee clean state between tests
  • ✓Use Cookie.Builder when setting domain, path, or expiry — the two-arg new Cookie(name, value) constructor leaves these null (browser defaults apply)
  • ✓Store session cookies in a Set<Cookie> variable after first login — re-inject them in subsequent tests to avoid repeated UI login flows

Interview Prep — 5 SDET Q&A

Q1. Why do you need to navigate to a domain before adding a cookie in Selenium?
A: Browsers enforce the same-origin policy — a cookie must belong to the currently loaded domain. Selenium mirrors this: calling addCookie() before driver.get(domainUrl) throws InvalidCookieDomainException. Always load the domain first, even if just the root URL.
Q2. What is the difference between deleteCookie(Cookie), deleteCookieNamed(String), and deleteAllCookies()?
A: deleteCookie(cookieObj) removes by matching the Cookie object's name. deleteCookieNamed("name") is a shortcut — removes by name string directly without needing the object. deleteAllCookies() removes every cookie for the current domain.
Q3. How would you use cookies to speed up test login?
A: Perform a real UI login once in a @BeforeSuite method, save the resulting cookies via getCookies() into a Set<Cookie>. In each subsequent test's @BeforeMethod, navigate to the domain and inject all saved cookies via addCookie(), then refresh. This bypasses the login page entirely.
Q4. What is Cookie.Builder and when do you use it over the simple constructor?
A: The simple new Cookie(name, value) constructor creates a minimal cookie with browser-default domain, path, and no expiry. Cookie.Builder lets you explicitly set all attributes: domain(), path(), expiresOn(), isSecure(), isHttpOnly(). Use Builder when testing cookie attributes or injecting production-like cookies for auth.
Q5. What does isHttpOnly() mean on a cookie, and how can you test it?
A: An httpOnly cookie cannot be read by JavaScript — it's a security measure against XSS attacks. In Selenium, you set it via Cookie.Builder.isHttpOnly(true) and verify it via cookie.isHttpOnly(). Testing: add cookie with isHttpOnly(true), call getCookieNamed() and assert cookie.isHttpOnly() == true.

Overview — Broken links: capture & validate

A broken link is a hyperlink on a webpage that returns an HTTP error response (4xx/5xx) instead of a valid page (2xx). Selenium itself cannot check HTTP status codes — it only controls the browser. We combine Selenium (to collect all <a> href URLs) with Java's HttpURLConnection (to send HTTP HEAD requests and check response codes).

What / Why / When / Where / How

WhatA link whose URL returns HTTP 4xx (client error) or 5xx (server error) — not accessible to users
WhyBroken links damage SEO, user experience, and indicate deployment or content issues
WhenAfter every deployment, as part of smoke tests, during regression runs on content-heavy sites
WhereOn home pages, nav menus, footer links, sitemap pages — high-impact link locations
HowSelenium collects hrefs → Java HttpURLConnection sends HEAD request → check response code ≥ 400

Keyword Glossary

HttpURLConnection
Java class to open HTTP connections and read response codes — no browser needed
getResponseCode()
Returns integer HTTP status code: 200=OK, 301=redirect, 404=not found, 500=server error
HEAD request
HTTP method that asks server for headers only — no body downloaded, much faster than GET
findElements(By.tagName)
Gets all anchor <a> tags on the page — returns List<WebElement>
getAttribute("href")
Reads the href attribute value from an <a> element — the actual URL string
setRequestMethod
Sets HTTP method on HttpURLConnection: "HEAD", "GET", "POST"
connect()
Opens the actual HTTP connection to the server — must be called before getResponseCode()
disconnect()
Releases the HTTP connection — always call in finally block to free resources
HTTP 200
OK — link is working. Also valid: 201 Created, 204 No Content, 301/302 Redirects
HTTP 4xx / 5xx
404 Not Found, 403 Forbidden, 500 Server Error = broken link categories

Interview one-liners

Selenium cannot check HTTP status codes directly — combine it with Java's HttpURLConnection to validate links.
Use HEAD requests instead of GET when checking links — it retrieves headers only with no body download, making it much faster.
A link with response code >= 400 is considered broken — 404 Not Found and 500 Server Error are the most common.
getAttribute("href") may return null for anchor tags without href or with javascript: URLs — always null-check before sending HTTP request.
Always call disconnect() on HttpURLConnection in a finally block to release the TCP connection and avoid resource leaks.

CollectAllLinksDemo.java
 1// ── Step 1: Collect all <a> hrefs from a webpage ─────────────────
 2// findElements(By.tagName("a")) → gets every <a> element on the page
 3// getAttribute("href") → extracts the URL from each anchor tag
 4// Result: a clean List<String> of all URLs ready for HTTP checking
 5import org.openqa.selenium.By;
 6import org.openqa.selenium.WebDriver;
 7import org.openqa.selenium.WebElement;
 8import org.openqa.selenium.chrome.ChromeDriver;
 9import java.util.ArrayList;
10import java.util.List;
11
12public class CollectAllLinksDemo {
13  public static void main(String[] args) {
14    WebDriver driver = new ChromeDriver();
15    driver.get("https://the-internet.herokuapp.com");
16
17    // Get ALL anchor (<a>) elements on the page (includes nav, footer, body)
18    List<WebElement> anchors = driver.findElements(By.tagName("a"));
19    System.out.println("Total <a> tags found: " + anchors.size());
20
21    // Extract href attribute from each <a> element
22    // IMPORTANT: href can be null (<a name="anchor">), empty (""),
23    // or a javascript: link — all must be filtered out
24    List<String> allUrls = new ArrayList<>();
25    for (WebElement a : anchors) {
26      String href = a.getAttribute("href"); // may return null
27      // Filter: skip null, blank, fragment-only (#), and javascript: links
28      if (href != null && !href.isEmpty()
29          && !href.startsWith("#")
30          && !href.startsWith("javascript:")
31          && !href.startsWith("mailto:")) {
32        allUrls.add(href);
33      }
34    }
35    System.out.println("Valid URLs to check: " + allUrls.size());
36    allUrls.forEach(System.out::println); // print all collected URLs
37    driver.quit();
38  }
39}

 1// ── Check a single URL with HttpURLConnection ─────────────────────

21    // setReadTimeout    → max time to wait for server response (ms)

24
25    // connect() → opens actual TCP connection to the server
26    conn.connect();
27
28    // getResponseCode() → returns HTTP status integer: 200, 301, 404, 500...
29    int code = conn.getResponseCode();
30    String status = (code >= 400) ? "BROKEN" : "VALID";
31    System.out.printf("[%s] HTTP %d → %s%n", status, code, urlStr);
32
33    // disconnect() → releases the TCP connection — ALWAYS call this
34    conn.disconnect();
35  }
36}

 2// THE CORE PATTERN: Selenium collects URLs, Java checks each one
 3import org.openqa.selenium.By;

 5import org.openqa.selenium.WebElement;
 6import org.openqa.selenium.chrome.ChromeDriver;
 7import java.net.HttpURLConnection;
 8import java.net.URL;
 9import java.util.List;
10
11public class FullPageLinkScanDemo {
12  public static void main(String[] args) {
13    WebDriver driver = new ChromeDriver();
14    driver.get("https://the-internet.herokuapp.com");
15
16    // PHASE 1: Selenium collects ALL <a> href URLs from the page
17    List<WebElement> links = driver.findElements(By.tagName("a"));
18    System.out.println("=== BROKEN LINK SCAN ===");
19    System.out.println("Total links found: " + links.size());
20
21    // PHASE 2: Check each link's HTTP response code
22    int brokenCount = 0, validCount = 0;
23    for (WebElement link : links) {
24      String href = link.getAttribute("href");
25      if (href == null || href.isEmpty() || href.startsWith("#")
26          || href.startsWith("javascript:")) continue; // skip invalid
27      HttpURLConnection conn = null;
28      try {
29        conn = (HttpURLConnection) new URL(href).openConnection();
30        conn.setRequestMethod("HEAD");
31        conn.setConnectTimeout(5000);
32        conn.connect();

HeadVsGetDemo.java
 1// ── HEAD vs GET request for link validation ───────────────────────
 2// HEAD is preferred but some servers return 405 (Method Not Allowed)
 3// Fallback pattern: try HEAD first → if 405, retry with GET
 4import java.net.HttpURLConnection;
 5import java.net.URL;
 6
 7public class HeadVsGetDemo {
 8
 9  // Smart link checker: tries HEAD, falls back to GET if HEAD not supported
10  static int checkUrl(String urlStr) {
11    HttpURLConnection conn = null;
12    try {
13      conn = (HttpURLConnection) new URL(urlStr).openConnection();
14      conn.setRequestMethod("HEAD"); // TRY HEAD first — faster
15      conn.setConnectTimeout(5000);
16      conn.connect();
17      int code = conn.getResponseCode();
18
19      // HTTP 405 = Method Not Allowed — server doesn't support HEAD
20      // Retry with GET to get the actual status code
21      if (code == 405) {
22        conn.disconnect();
23        conn = (HttpURLConnection) new URL(urlStr).openConnection();
24        conn.setRequestMethod("GET"); // FALLBACK to GET
25        conn.setConnectTimeout(5000);
26        conn.connect();
27        code = conn.getResponseCode();
28        System.out.println("  (HEAD→405, retried with GET)");
29      }
30      return code;
31    } catch (Exception e) { return -1; }
32    finally { if (conn != null) conn.disconnect(); }
33  }
34
35  public static void main(String[] args) {
36    String[] urls = {
37      "https://the-internet.herokuapp.com",
38      "https://the-internet.herokuapp.com/status_codes/404",
39      "https://the-internet.herokuapp.com/status_codes/500"
40    };
41    for (String url : urls) {
42      int code = checkUrl(url);
43      System.out.printf("HTTP %-6d [%s] %s%n", code,
44        code >= 400 ? "BROKEN" : "VALID ", url);
45    }
46  }

Best Practices

  • ✓Always use setRequestMethod("HEAD") first — it downloads headers only (no body) making it 2–5x faster than GET for large pages
  • ✓Implement HEAD-to-GET fallback: some servers return 405 Method Not Allowed for HEAD — retry with GET in that case
  • ✓Always wrap the connection check in try-catch-finally — conn.disconnect() in finally prevents TCP connection leaks
  • ✓Filter out null, #, javascript:, mailto:, tel: hrefs before sending HTTP requests — they will throw MalformedURLException
  • ✓Set reasonable timeouts (setConnectTimeout(5000)) — some servers are slow and will hang your entire test if no timeout is set
  • ✓Consider 3xx redirects (301, 302) as valid by default — only flag them if you specifically need to detect outdated URLs

Interview Prep — 5 SDET Q&A

Q1. Can Selenium alone detect broken links? Why or why not?
A: No. Selenium automates the browser UI — it clicks, types, and reads elements. It cannot access HTTP response codes because those happen at the network layer before the browser renders anything. We must combine Selenium (to collect href values) with Java's HttpURLConnection (to make HTTP requests and read status codes).
Q2. What is an HTTP HEAD request and why is it preferred for link checking?
A: HEAD is an HTTP method that asks the server to return only the response headers — no body/content. This means we get the status code (200, 404, 500) without downloading the full HTML/CSS/images of the page. This makes link checking 2–5x faster than using GET, which downloads the full response body.
Q3. What HTTP codes indicate a broken link?
A: Any response code >= 400. Common broken link codes: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found (most common), 410 Gone, 500 Internal Server Error, 503 Service Unavailable. Codes 200–399 (including 301/302 redirects) are generally considered valid.
Q4. Why must you always call disconnect() on HttpURLConnection?
A: HttpURLConnection uses a TCP connection from the JVM's connection pool. Without disconnect(), the connection stays open and is counted against the OS's maximum open socket limit. In a loop checking hundreds of links, not disconnecting causes resource exhaustion, connection refused errors, and test hangs.
Q5. How would you handle links that return null from getAttribute("href")?
A: An anchor tag without a href attribute (e.g., <a name="top"> or <a> acting as a placeholder) returns null from getAttribute("href"). Always null-check before using the value. In the filtering step: if (href == null || href.isEmpty()) continue; — skipping these safely without causing NullPointerException.

ParallelLinkCheckerDemo.java
 1// ── Parallel link checking using ExecutorService ───────────────────
 2// ExecutorService → Java thread pool manager (java.util.concurrent)
 3// Callable → task that returns a result (here: HTTP response code)
 4// Future   → placeholder for a result that will arrive from a thread
 5import org.openqa.selenium.By;
 6import org.openqa.selenium.WebDriver;
 7import org.openqa.selenium.chrome.ChromeDriver;
 8import java.net.HttpURLConnection;
 9import java.net.URL;
10import java.util.*;
11import java.util.concurrent.*; // ExecutorService, Callable, Future
12
13public class ParallelLinkCheckerDemo {
14  public static void main(String[] args) throws Exception {
15    WebDriver driver = new ChromeDriver();
16    driver.get("https://the-internet.herokuapp.com");
17
18    // Collect all valid HTTP hrefs from the page
19    List<String> urls = new ArrayList<>();
20    driver.findElements(By.tagName("a")).forEach(a -> {
21      String h = a.getAttribute("href");
22      if (h != null && h.startsWith("http")) urls.add(h);
23    });
24    driver.quit();
25    System.out.println("Checking " + urls.size() + " links in PARALLEL...");
26
27    // newFixedThreadPool(10) → creates pool of 10 concurrent threads
28    // 10 threads check 10 links simultaneously instead of 1 at a time
29    ExecutorService pool = Executors.newFixedThreadPool(10);
30
31    // Build a Callable task for each URL — Callable returns a result
32    List<Callable<String>> tasks = new ArrayList<>();
33    for (String url : urls) {
34      tasks.add(() -> {       // lambda creates a Callable for each URL
35        HttpURLConnection c = (HttpURLConnection) new URL(url).openConnection();
36        try {
37          c.setRequestMethod("HEAD"); c.setConnectTimeout(5000); c.connect();
38          int code = c.getResponseCode();
39          return (code >= 400 ? "[BROKEN] " : "[VALID]  ") + code + " " + url;
40        } finally { c.disconnect(); }
41      });
42    }
43    // invokeAll() → submits ALL tasks and blocks until ALL complete
44    List<Future<String>> results = pool.invokeAll(tasks);
45    // future.get() → retrieves the result of each completed task
46    for (Future<String> f : results) System.out.println(f.get());
47    pool.shutdown(); // shutdown() → no new tasks, wait for current to finish
48  }
49}

24      for (var a : driver.findElements(By.tagName("a"))) {
25        String href = a.getAttribute("href"), txt = a.getText().trim();
26        if (href == null || !href.startsWith("http")) continue;
27        HttpURLConnection c = null; int code = 0; String css = "V";
28        try {
29          c = (HttpURLConnection) new URL(href).openConnection();
30          c.setRequestMethod("HEAD"); c.setConnectTimeout(4000); c.connect();
31          code = c.getResponseCode();
32          css  = code >= 400 ? "B" : "V";
33        } catch (Exception e) { css = "E"; code = -1; }
34        finally { if (c != null) c.disconnect(); }
35        fw.write(String.format("<tr class='%s'><td>%s</td><td>%d</td><td>%s</td><td>%s</td></tr>",
36          css, css.equals("V") ? "VALID" : (css.equals("B") ? "BROKEN" : "ERROR"),
37          code, href, txt));
38      }
39      fw.write("</table></body></html>");
40    }
41    System.out.println("HTML report saved: link-report.html");
42    driver.quit();
43  }
44}

 5import org.openqa.selenium.WebDriver;
 6import org.openqa.selenium.WebElement;
 7import java.net.HttpURLConnection;
 8import java.net.URL;
 9import java.util.ArrayList;
10import java.util.List;
11
12public class LinkCheckerUtil {
13
14  // getHttpCode(url) → returns HTTP response code for a URL
15  // Returns -1 on connection error
16  public static int getHttpCode(String url) {
17    HttpURLConnection conn = null;
18    try {
19      conn = (HttpURLConnection) new URL(url).openConnection();
20      conn.setRequestMethod("HEAD"); conn.setConnectTimeout(5000); conn.connect();
21      int code = conn.getResponseCode();
22      if (code == 405) { // HEAD not supported — retry with GET
23        conn.disconnect();
24        conn = (HttpURLConnection) new URL(url).openConnection();
25        conn.setRequestMethod("GET"); conn.setConnectTimeout(5000); conn.connect();
26        return conn.getResponseCode();
27      }
28      return code;
29    } catch (Exception e) { return -1; }
30    finally { if (conn != null) conn.disconnect(); }
31  }
32
33  // isBroken(code) → true if code indicates a broken/unreachable link
34  public static boolean isBroken(int code) { return code >= 400 || code == -1; }
35
36  // getBrokenLinks(driver) → returns List of all broken href URLs on page
37  public static List<String> getBrokenLinks(WebDriver driver) {
38    List<String> broken = new ArrayList<>();
39    for (WebElement a : driver.findElements(By.tagName("a"))) {
40      String href = a.getAttribute("href");
41      if (href == null || !href.startsWith("http")) continue;
42      if (isBroken(getHttpCode(href))) broken.add(href);
43    }
44    return broken;
45  }
46
47  // getBrokenImages(driver) → returns List of broken <img src> URLs
48  public static List<String> getBrokenImages(WebDriver driver) {
49    List<String> broken = new ArrayList<>();
50    for (WebElement img : driver.findElements(By.tagName("img"))) {
51      String src = img.getAttribute("src");
52      if (src != null && src.startsWith("http") && isBroken(getHttpCode(src))) broken.add(src);
53    }
54    return broken;
55  }
56}

Best Practices

  • ✓Use parallel checking (ExecutorService with 10–20 threads) for pages with 50+ links — reduces scan time from minutes to seconds
  • ✓Build a reusable LinkCheckerUtil class — keeps HTTP logic out of test methods and follows the Single Responsibility Principle
  • ✓Check image src attributes as well as anchor href attributes — broken images are as harmful as broken links to users
  • ✓Use scoped findElements on a section WebElement when you only want to check links in nav/footer/header areas
  • ✓Generate an HTML report file (link-report.html) with colour-coded results — easier to share with developers than console output
  • ✓Shut down the ExecutorService via pool.shutdown() after invokeAll() — otherwise JVM won't exit cleanly after tests

Interview Prep — 5 SDET Q&A

Q1. Why use ExecutorService for broken link checking instead of a simple loop?
A: A sequential loop checks one link at a time — each HTTP request typically takes 200–500ms. For 100 links that's 20–50 seconds. An ExecutorService with 10 threads checks 10 links simultaneously, reducing total time to ~5 seconds. The tradeoff: thread-safety must be considered for shared data structures (use Collections.synchronizedList() or CopyOnWriteArrayList).
Q2. How do you check broken links in a specific section (e.g., only the footer)?
A: Instead of calling driver.findElements(By.tagName("a")) (searches the whole page), first locate the section element: WebElement footer = driver.findElement(By.tagName("footer")), then call footer.findElements(By.tagName("a")) — this returns only anchors within that specific section.
Q3. How would you detect redirect chains in link validation?
A: Set conn.setInstanceFollowRedirects(false) before connecting. The response code will then show the raw 301/302 instead of the final 200. Read conn.getHeaderField("Location") to get the redirect target URL. This lets you flag outdated URLs that redirect and optionally check the final destination separately.
Q4. How do you check broken images with Selenium?
A: Use driver.findElements(By.tagName("img")) to get all image elements, then img.getAttribute("src") to get the image URL. Send an HTTP HEAD request to each src URL via HttpURLConnection. Response code >= 400 means the image is broken and will show as a missing image icon in the browser.
Q5. Where in a test framework would you place a broken link utility class?
A: In a utils/ or helpers/ package as a static utility class (e.g., com.myframework.utils.LinkCheckerUtil). It should have no dependencies on test frameworks or page object classes — only WebDriver and java.net. This makes it reusable across all test classes and easy to unit-test independently.
