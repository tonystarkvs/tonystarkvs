Selenium Java: Complete Learning Hub

Selenium is a free, open-source browser automation framework that lets you control web browsers programmatically. Think of it as a "robot hand" that types, clicks, and navigates websites exactly like a human — but automated, fast, and repeatable. It is used for functional testing, regression testing, and UI automation of web applications.

🎙 Interview one-liners

Selenium is an open-source tool used to automate web browser interactions for testing purposes.
Selenium supports multiple browsers like Chrome, Firefox, Edge, and Safari via their native drivers.
Selenium is not a testing framework — it is a browser automation library used alongside frameworks like TestNG or JUnit.
Selenium supports multiple languages: Java, Python, C#, JavaScript, Ruby, and Kotlin.
Selenium WebDriver directly communicates with the browser without any intermediate server in v3, but v4 uses W3C protocol natively.
Selenium is platform-independent and can run on Windows, Linux, and macOS environments.

// ─── CODE EXAMPLE 1: Your very first Selenium WebDriver program ───
// 'import' keyword → tells Java where to find the class definition
// 'org.openqa.selenium' → this is the Selenium package (library namespace)
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

// 'public class' → declares a class visible to all
// A class is a blueprint — think of it as a container for your code
public class FirstSeleniumTest {

    // 'public static void main' → entry point of any Java program
    // 'static' means no object of this class is needed to call this method
    public static void main(String[] args) {

        // WebDriver → interface (like a contract) defining all browser actions
        // ChromeDriver → class that implements WebDriver for Chrome browser
        // 'new' keyword → creates a fresh instance (object) of ChromeDriver
        WebDriver driver = new ChromeDriver();

        // driver.get() → opens the given URL in the browser
        // This is equivalent to typing in the address bar and pressing Enter
        driver.get("https://www.google.com");

        // driver.getTitle() → returns the text in the browser tab (page title)
        System.out.println("Page title: " + driver.getTitle());

        // driver.quit() → closes ALL browser windows AND ends the WebDriver session
        // ALWAYS call quit() to free memory — failing to do so causes resource leaks
        driver.quit();
    }
}

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

        options.addArguments("--start-maximized");

        WebDriver driver = new ChromeDriver(options);

        // Use close() when you have multiple tabs open

    }
}

// ─── CODE EXAMPLE 5: Window Management ───
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.Dimension;

public class WindowManagement {
    public static void main(String[] args) {

        WebDriver driver = new ChromeDriver();
        driver.get("https://www.example.com");

        // manage().window() → returns a Window object to control size/position
        // maximize() → expands browser to fill the entire screen
        driver.manage().window().maximize();

        // minimize() → shrinks browser to taskbar (Selenium 4 feature)
        driver.manage().window().minimize();

        // fullscreen() → true fullscreen mode (hides OS taskbar)
        driver.manage().window().fullscreen();

        // Dimension → a class holding width and height as integers
        // setSize() → resizes browser to custom pixel dimensions
        // Useful for responsive/mobile testing
        Dimension d = new Dimension(1280, 720);
        driver.manage().window().setSize(d);

        System.out.println("Window size: " + driver.manage().window().getSize());

        driver.quit();
    }
}

// ─── CODE EXAMPLE 6: Implicit Wait — Handle slow-loading pages ───
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import java.time.Duration; // Duration → standard Java class for time values

public class ImplicitWaitDemo {
    public static void main(String[] args) {

        WebDriver driver = new ChromeDriver();

        // Implicit Wait → tells WebDriver: "if element not found immediately,
        // keep POLLING (checking repeatedly) for up to N seconds before failing"
        // Duration.ofSeconds(10) → 10 second maximum wait time
        // This applies GLOBALLY to all findElement() calls in this session
        driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

        driver.get("https://www.example.com");

        // Without the wait, if page is slow, this line would throw NoSuchElementException
        // With implicit wait, Selenium waits up to 10s for the element to appear
        driver.findElement(By.tagName("h1"));

        System.out.println("Element found successfully!");
        driver.quit();
    }
}

✅ Best Practices

  • Always call driver.quit() in a finally block — ensures browser closes even if test fails
  • Never hardcode browser driver paths — use WebDriverManager or Selenium Manager (v4.6+)
  • Set implicit wait once at test setup, not inside individual test methods
  • Use ChromeOptions for headless mode in CI/CD environments (no UI needed)
  • Always assert the page title or URL after navigation to confirm the right page loaded
  • Use interface type WebDriver driver (not ChromeDriver driver) for flexibility across browsers

🎯 SDET Interview Q&A

Q1. What is the difference between driver.close() and driver.quit()?
A: close() closes only the currently focused browser tab/window. quit() closes ALL browser windows and terminates the entire WebDriver session, freeing all resources.
Q2. What is WebDriver in Selenium?
A: WebDriver is an interface (API) in Selenium that provides methods to interact with a browser. Classes like ChromeDriver and FirefoxDriver implement this interface for specific browsers.
Q3. Is Selenium a testing framework?
A: No. Selenium is a browser automation library. For test management (assertions, reports, test structure), you combine it with TestNG or JUnit frameworks.
Q4. What languages does Selenium support?
A: Selenium officially supports Java, Python, C#, JavaScript (Node.js), Ruby, and Kotlin with maintained client bindings.
Q5. What happens if you don't call quit() after tests?
A: Browser processes remain open in the OS memory, ChromeDriver orphan processes accumulate, causing memory leaks and test instability in subsequent runs.

Overview — Selenium Architecture

Selenium's architecture defines HOW your Java code talks to the browser. In Selenium 3, it used the JSON Wire Protocol. In Selenium 4, it uses the native W3C WebDriver Protocol — a standardized HTTP-based communication specification. Understanding this flow is critical for any SDET interview.

🎙 Interview one-liners

Selenium 4 uses the W3C WebDriver standard protocol, making it more stable and browser-compatible.
In Selenium's architecture, the Java client sends HTTP requests to the browser driver, which forwards commands to the actual browser.
ChromeDriver acts as a bridge (middleware) between Selenium WebDriver and the Chrome browser.
W3C protocol eliminates the need for JSON Wire Protocol translation layer used in Selenium 3.
Each browser has its own driver: ChromeDriver for Chrome, GeckoDriver for Firefox, EdgeDriver for Edge.
Selenium Grid extends this architecture to run tests remotely on different machines and browsers simultaneously.

// ─── CODE EXAMPLE 1: Understanding W3C Protocol in Selenium 4 ───
// Selenium 4 uses W3C WebDriver standard natively (no translation needed)
// The protocol sends JSON commands over HTTP to the browser driver

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.chrome.ChromeOptions;

public class W3CProtocolDemo {
    public static void main(String[] args) {

        // ChromeOptions → W3C-compliant capability container
        // In Selenium 4, ALL capabilities must go through Options objects
        // This replaces the old DesiredCapabilities approach (deprecated)
        ChromeOptions options = new ChromeOptions();

        // setCapability() → sets W3C standard capability key-value pairs
        // W3C capabilities are standardized — same names across all browsers
        options.setCapability("browserName", "chrome");
        options.setCapability("browserVersion", "latest");

        // When you do 'new ChromeDriver(options)', Selenium internally:
        // 1. Starts ChromeDriver.exe process
        // 2. Sends POST /session (HTTP request) with capabilities JSON
        // 3. ChromeDriver launches Chrome and returns a session ID
        WebDriver driver = new ChromeDriver(options);

        driver.get("https://www.selenium.dev");
        System.out.println("W3C Session active: " + driver.getTitle());

        driver.quit();
    }
}

// ─── CODE EXAMPLE 2: Remote WebDriver — Architecture in Action ───
// RemoteWebDriver → sends commands to a REMOTE machine (not local)
// Used with Selenium Grid or cloud services like BrowserStack/Sauce Labs

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.remote.RemoteWebDriver;
import org.openqa.selenium.chrome.ChromeOptions;
import java.net.URL;

public class RemoteDriverDemo {
    public static void main(String[] args) throws Exception {

        // ChromeOptions → browser config to send to remote node
        ChromeOptions options = new ChromeOptions();
        options.addArguments("--headless"); // headless = no UI (runs silently)

        // URL → the address of the Selenium Grid Hub or Remote server
        // 4444 → default port Selenium Grid Hub listens on
        URL gridUrl = new URL("http://localhost:4444");

        // RemoteWebDriver → sends HTTP commands to the remote Hub
        // Hub then routes to an appropriate Node machine
        WebDriver driver = new RemoteWebDriver(gridUrl, options);

        driver.get("https://www.google.com");
        System.out.println("Remote Title: " + driver.getTitle());

        driver.quit(); // closes remote browser session
    }
}

// ─── CODE EXAMPLE 5: Running Multiple Browsers Simultaneously ───
// Architecture allows multiple WebDriver instances (sessions) at once

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.firefox.FirefoxDriver;
import org.openqa.selenium.edge.EdgeDriver;

public class MultiBrowserDemo {
    public static void main(String[] args) {

        // Each 'new XDriver()' creates a SEPARATE WebDriver session
        // Each session is independent — has its own cookies, tabs, history
        WebDriver chrome  = new ChromeDriver();
        WebDriver firefox = new FirefoxDriver();
        WebDriver edge    = new EdgeDriver();

        // All three browsers open the same URL simultaneously
        String url = "https://www.example.com";
        chrome.get(url);
        firefox.get(url);
        edge.get(url);

        // Print titles from all browsers
        System.out.println("Chrome:  " + chrome.getTitle());
        System.out.println("Firefox: " + firefox.getTitle());
        System.out.println("Edge:    " + edge.getTitle());

        // Always quit each driver separately
        chrome.quit();
        firefox.quit();
        edge.quit();
    }
}

  • Always use W3C-compliant Options classes (ChromeOptions, FirefoxOptions) instead of deprecated DesiredCapabilities
  • Run java -jar selenium-server.jar standalone for local Grid testing without needing Hub+Node setup

  • Always verify browser driver version matches installed browser version to prevent session creation failures

🎯 SDET Interview Q&A

Q1. What protocol does Selenium 4 use and why is it better than Selenium 3?
A: Selenium 4 natively uses W3C WebDriver protocol, eliminating the JSON Wire Protocol translation layer from v3. This means fewer compatibility issues and more stable cross-browser behavior.
Q2. What is the role of ChromeDriver in Selenium architecture?
A: ChromeDriver is an executable that acts as a bridge between Selenium's HTTP commands and Chrome's DevTools Protocol (CDP). It translates WebDriver commands into Chrome-understandable instructions.
Q3. What is Selenium Grid and when do you use it?
A: Selenium Grid allows parallel test execution across multiple machines and browsers. Use it when you need to cut test execution time via parallelism or need cross-browser/cross-OS testing on different physical/virtual machines.
Q4. What changed in Selenium Grid 4 vs Grid 3?
A: Grid 4 introduced a new architecture: Router, Distributor, SessionMap, and Node. It also supports Docker, Kubernetes, and has a built-in GraphQL API for monitoring sessions.
Q5. How does RemoteWebDriver differ from ChromeDriver?
A: ChromeDriver runs locally on your machine. RemoteWebDriver sends WebDriver commands over HTTP to a remote machine (Grid Hub), enabling execution on remote/cloud infrastructure.

Overview — Selenium Components

Selenium is not a single tool — it is a suite of 4 components, each serving a different purpose. Understanding what each component does, when to use it, and how they relate to each other is foundational knowledge for any automation engineer.

🎙 Interview one-liners

Selenium WebDriver is the core component for programmatic browser control using code.
Selenium IDE is a browser plugin for record-and-playback test creation — no coding required.
Selenium Grid enables parallel test execution across multiple machines and browser configurations.
Selenium RC (Remote Control) is the predecessor to WebDriver and is now fully deprecated.
Selenium WebDriver replaced Selenium RC because it communicates directly with browsers without a proxy server.
The Selenium Manager (introduced in v4.6) auto-manages browser driver binaries without manual setup.

🧩 4 Components Visual

Selenium WebDriver
Core library. Controls browser via HTTP+W3C. Write Java/Python code to interact with DOM elements. Primary tool for all professional automation.
Selenium IDE
Browser extension (Chrome/Firefox). Records your clicks/typing and replays them. Exports to Java, Python. Good for quick prototyping or non-coders.
Selenium Grid
Distributed execution server. Runs tests on multiple machines/browsers in parallel. Reduces total test suite execution time drastically.
Selenium RC (Legacy)
Fully deprecated. Was the original Selenium automation approach using a JavaScript injection proxy server. Not used in any modern project.

// ─── CODE EXAMPLE 1: findElement() — Locate a single web element ───
// 'WebElement' → interface representing a single HTML element on the page
// Think of it as a Java handle to a button, input, link, or any HTML tag

import org.openqa.selenium.By;       // By → class providing locator strategies
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement; // WebElement → interface for DOM element
import org.openqa.selenium.chrome.ChromeDriver;

public class FindElementDemo {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.google.com");

        // findElement() → returns the FIRST matching element
        // Throws NoSuchElementException if element not found
        // By.name() → locates element by its HTML 'name' attribute
        // <input name="q"> is Google's search box
        WebElement searchBox = driver.findElement(By.name("q"));

        // sendKeys() → types text into an input field
        // Like a human typing on the keyboard
        searchBox.sendKeys("Selenium Java Tutorial");

        // submit() → submits the form containing this element
        searchBox.submit();

        System.out.println("Search submitted. Title: " + driver.getTitle());
        driver.quit();
    }
}

// ─── CODE EXAMPLE 2: findElements() — Get multiple elements at once ───
// findElements() (plural) → returns a List of all matching WebElements
// Returns EMPTY LIST (not exception) if no elements found

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import java.util.List; // List → Java ordered collection interface

public class FindElementsDemo {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.selenium.dev/documentation/");

        // By.tagName("a") → finds ALL anchor (<a>) tags on the page
        List<WebElement> allLinks = driver.findElements(By.tagName("a"));

        // .size() → number of elements in the List
        System.out.println("Total links found: " + allLinks.size());

        // Enhanced for-loop to iterate over every WebElement in the list
        for (WebElement link : allLinks) {
            // getText() → returns visible text content of the element
            // getAttribute("href") → returns value of 'href' HTML attribute
            System.out.println(link.getText() + " → " + link.getAttribute("href"));
        }

        driver.quit();
    }
}

// ─── CODE EXAMPLE 3: All 8 By Locator Strategies ───
// By class provides 8 ways to find elements — choose the most stable one

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;

public class LocatorStrategies {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.example.com");

        // 1. By.id() — FASTEST and most reliable (unique per page per HTML spec)
        driver.findElement(By.id("username"));

        // 2. By.name() — uses HTML 'name' attribute (common in forms)
        driver.findElement(By.name("email"));

        // 3. By.className() — matches HTML class attribute (can return multiple)
        driver.findElement(By.className("btn-primary"));

        // 4. By.tagName() — matches HTML tag type (e.g., h1, input, button)
        driver.findElement(By.tagName("h1"));

        // 5. By.linkText() — exact full text of a clickable <a> link
        driver.findElement(By.linkText("More information..."));

        // 6. By.partialLinkText() — partial match of link text
        driver.findElement(By.partialLinkText("More"));

        // 7. By.cssSelector() — CSS selector syntax (powerful, widely used)
        driver.findElement(By.cssSelector("input[type='text']"));

        // 8. By.xpath() — XPath query (most powerful but slower)
        driver.findElement(By.xpath("//input[@id='search']"));

        driver.quit();
    }
}

// ─── CODE EXAMPLE 4: Actions Class — Mouse & Keyboard Interactions ───
// Actions → class for complex user gestures: hover, drag, right-click, etc.

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.interactions.Actions; // Actions → complex gesture class

public class ActionsClassDemo {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.example.com");

        // Create an Actions object — needs driver as parameter
        Actions actions = new Actions(driver);

        WebElement menu = driver.findElement(By.id("nav-menu"));

        // moveToElement() → simulates mouse HOVER over an element
        // perform() → EXECUTES the built-up action chain
        actions.moveToElement(menu).perform();

        // contextClick() → right-click on an element (opens context menu)
        WebElement item = driver.findElement(By.id("item"));
        actions.contextClick(item).perform();

        // doubleClick() → double-click (used for file rename, text select, etc.)
        actions.doubleClick(item).perform();

        // dragAndDrop(source, target) → drag element from one place to another
        WebElement source = driver.findElement(By.id("drag"));
        WebElement target = driver.findElement(By.id("drop"));
        actions.dragAndDrop(source, target).perform();

        driver.quit();
    }
}

// ─── CODE EXAMPLE 5: JavascriptExecutor — Direct JS execution ───
// JavascriptExecutor → interface to run JavaScript in the browser context
// Used when WebDriver can't interact with hidden/custom elements

import org.openqa.selenium.JavascriptExecutor; // Key interface
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.By;
import org.openqa.selenium.WebElement;

public class JavaScriptDemo {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.google.com");

        // Cast driver to JavascriptExecutor to access JS methods
        // 'Casting' = treating an object as a different (related) type
        JavascriptExecutor js = (JavascriptExecutor) driver;

        // executeScript() → runs JS in the browser and returns result
        // "return document.title" → JS expression to get page title
        Object title = js.executeScript("return document.title;");
        System.out.println("JS Title: " + title);

        // Scroll the page by 500 pixels downward using JS
        js.executeScript("window.scrollBy(0, 500);");

        // Click an element using JS (when regular .click() fails)
        WebElement btn = driver.findElement(By.name("q"));
        js.executeScript("arguments[0].click();", btn);
        // 'arguments[0]' → refers to the first Java object passed (btn)

        driver.quit();
    }
}

✅ Best Practices

  • Prefer By.id() and By.cssSelector() over XPath — they are faster and more readable
  • Use findElements() (plural) to check element existence (returns empty list, not exception)
  • Always use Actions class for hover/drag — don't use JavaScript unless WebDriver can't handle it
  • Use JavascriptExecutor as a last resort for elements not interactable via standard WebDriver
  • Capture screenshots on test failure in @AfterMethod hooks for test evidence
  • Avoid XPath with absolute paths — they break if page structure changes even slightly

🎯 SDET Interview Q&A

Q1. What is the difference between findElement() and findElements()?
A: findElement() returns a single WebElement and throws NoSuchElementException if not found. findElements() returns a List<WebElement> and returns an empty list if nothing matches — no exception thrown.
Q2. Which locator strategy is best and why?
A: By.id() is fastest and most reliable per HTML spec (IDs must be unique). For complex scenarios, By.cssSelector() is preferred over XPath for speed and readability.
Q3. When would you use JavascriptExecutor?
A: When standard WebDriver interactions fail — e.g., clicking a hidden element, scrolling to an element, setting a readonly input value, or reading custom DOM properties.
Q4. What is the Actions class used for?
A: Complex user gestures: hover (mouse-over), right-click, double-click, drag-and-drop, keyboard combos (Ctrl+A), and slow typing. Regular .click() can't handle these.
Q5. Name all 4 Selenium components.
A: (1) Selenium WebDriver — core browser automation library; (2) Selenium IDE — record-and-playback browser extension; (3) Selenium Grid — distributed parallel execution server; (4) Selenium RC — legacy, fully deprecated.

Overview — Selenium v3 vs v4

Selenium 4 is not just a version bump — it is a complete architectural and API overhaul. Understanding what changed, what was removed, and what was added is essential for modern SDET roles. The shift from JSON Wire Protocol to W3C is the biggest fundamental change.

🎙 Interview one-liners

Selenium 4 fully adopted the W3C WebDriver standard, making it browser-vendor agnostic.
Selenium 4 introduced native CDP (Chrome DevTools Protocol) support for network interception.
Relative Locators (near, above, below, toLeftOf, toRightOf) are new in Selenium 4.
Selenium 4 deprecated DesiredCapabilities — replaced by browser-specific Options classes.
Selenium Grid 4 was completely rewritten with a new Hub-Node architecture using Docker support.
Selenium 4 eliminated the need for JSON Wire Protocol translation, reducing flakiness.

// ─── CODE EXAMPLE 5: CDP — Chrome DevTools Protocol (Selenium 4) ───
// CDP lets Selenium 4 control Chrome internals: network, console, performance
// This was IMPOSSIBLE with Selenium 3 without third-party tools

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.devtools.DevTools; // DevTools → CDP access interface
import org.openqa.selenium.devtools.v120.network.Network; // CDP Network domain
import java.util.Optional;

public class CDPDemo {
    public static void main(String[] args) {
        ChromeDriver driver = new ChromeDriver();
        // Note: Must use ChromeDriver (not WebDriver) to access getDevTools()

        // getDevTools() → returns DevTools object for CDP communication
        DevTools devTools = driver.getDevTools();

        // createSession() → opens a CDP session with Chrome's internal engine
        devTools.createSession();

        // Enable the Network domain — allows monitoring of HTTP requests
        devTools.send(Network.enable(
            Optional.empty(), Optional.empty(), Optional.empty()
        ));

        // addListener → callback when ANY network request is made
        devTools.addListener(Network.requestWillBeSent(), req -> {
            System.out.println("Request: " + req.getRequest().getUrl());
        });

        driver.get("https://www.selenium.dev");
        driver.quit();
    }
}

🎯 SDET Interview Q&A

Q1. What is the biggest change in Selenium 4?
A: Native W3C WebDriver protocol support. Selenium 3 used its own JSON Wire Protocol which required translation. Selenium 4 speaks W3C directly, eliminating compatibility issues between browser and driver versions.
Q2. What are Relative Locators?
A: New in Selenium 4 — locators that find elements based on their visual position relative to another element. Methods: near(), above(), below(), toLeftOf(), toRightOf(). Uses JavaScript's getBoundingClientRect() internally.
Q3. How do you open a new tab in Selenium 4?
A: driver.switchTo().newWindow(WindowType.TAB). In Selenium 3, you had to use JavascriptExecutor with window.open() command.
Q4. What is Selenium Manager?
A: Introduced in Selenium 4.6, it's a built-in binary that auto-downloads the correct browser driver (ChromeDriver, GeckoDriver, etc.) matching your installed browser version. Eliminates all System.setProperty setup code.
Q5. What is CDP support in Selenium 4?
A: Chrome DevTools Protocol integration lets Selenium 4 directly control Chrome internals — intercept network requests, mock geolocation, set device emulation, capture performance metrics, and block specific URLs. Not available in Selenium 3.

Overview — Deprecated in Selenium 4

When upgrading from Selenium 3 to 4, certain APIs were deprecated or removed. Using deprecated APIs causes compiler warnings and may break in future Selenium releases. Knowing these ensures you write future-proof automation code.

🎙 Interview one-liners

DesiredCapabilities is deprecated in Selenium 4 — use browser-specific Options classes instead.
FindsByXPath, FindsById and similar FindsBy interfaces were deprecated in favor of findElement(By).
The MutableCapabilities merge() chaining approach changed — mapCapabilities() API was refactored.
System.setProperty for driver paths is deprecated — Selenium Manager handles this automatically.
ExpectedConditions with Predicate was deprecated — use FluentWait directly.
The withTimeout() overload using TimeUnit is deprecated — use Duration objects instead.

⚠️ Deprecated APIs Quick Reference

DEPRECATED DesiredCapabilities→ Use ChromeOptions / FirefoxOptions / EdgeOptions
DEPRECATED implicitlyWait(long, TimeUnit)→ Use implicitlyWait(Duration.ofSeconds(n))
DEPRECATED pageLoadTimeout(long, TimeUnit)→ Use pageLoadTimeout(Duration.ofSeconds(n))
DEPRECATED setScriptTimeout(long, TimeUnit)→ Use scriptTimeout(Duration.ofSeconds(n))
DEPRECATED withTimeout(long, TimeUnit)→ Use withTimeout(Duration.ofSeconds(n))
DEPRECATED System.setProperty driver→ Selenium Manager auto-downloads drivers

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.support.ui.FluentWait;
import org.openqa.selenium.NoSuchElementException;
import java.time.Duration;

public class FluentWaitFixed {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.example.com");

        // ❌ DEPRECATED overload (Selenium 3 style):
        // new FluentWait<>(driver).withTimeout(10, TimeUnit.SECONDS)
        // .pollingEvery(500, TimeUnit.MILLISECONDS);

        // ✅ CORRECT Selenium 4 FluentWait:
        // FluentWait → generic wait that accepts driver and returns WebElement
        FluentWait<WebDriver> wait = new FluentWait<>(driver)

            .ignoring(NoSuchElementException.class); // ignore this exception while waiting

        // until() → repeatedly calls this lambda until it returns non-null/true
        WebElement element = wait.until(d ->
            d.findElement(By.tagName("h1"))
        );

        System.out.println("Found: " + element.getText());

import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.devtools.DevTools;
import org.openqa.selenium.devtools.v120.log.Log; // CDP Log domain

public class CDPConsoleCapture {
    public static void main(String[] args) {

        ChromeDriver driver = new ChromeDriver();
        DevTools devTools = driver.getDevTools();
        devTools.createSession();

        // Enable the Log domain — starts sending log events to Java
        devTools.send(Log.enable());

        // addListener → registers a callback for EVERY log entry
        // This fires whenever browser console produces output
        devTools.addListener(Log.entryAdded(), entry -> {
            // entry.getLevel() → severity: VERBOSE, INFO, WARNING, ERROR
            // entry.getText() → the actual console message
            // entry.getSource() → where the log came from (javascript, network, etc.)
            System.out.printf(
                "[%s] %s: %s%n",
                entry.getLevel(),
                entry.getSource(),
                entry.getText()
            );
        });

        driver.get("https://www.example.com");
        System.out.println("Console logs captured via CDP!");

        driver.quit();
    }
}

// ─── CODE EXAMPLE 3: CDP — Block Network Requests ───
// Use case: Test behavior when specific resources fail to load
// Block ads, images, analytics scripts during testing

import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.devtools.DevTools;
import org.openqa.selenium.devtools.v120.network.Network;
import org.openqa.selenium.devtools.v120.network.model.ResourceType;
import java.util.Optional;

public class CDPNetworkBlock {
    public static void main(String[] args) {

        ChromeDriver driver = new ChromeDriver();
        DevTools devTools = driver.getDevTools();
        devTools.createSession();

        // Enable Network domain to intercept requests
        devTools.send(Network.enable(
            Optional.empty(), Optional.empty(), Optional.empty()
        ));

        // setBlockedURLs() → list of URL patterns to block
        // Browser will fail any request matching these patterns
        // Useful to simulate slow/broken networks or block ads
        devTools.send(Network.setBlockedURLs(
            java.util.List.of(
                "*google-analytics*",  // block analytics
                "*doubleclick.net*",   // block ad network
                "*.png"                // block all PNG images
            )
        ));

        driver.get("https://www.example.com");
        System.out.println("Page loaded with blocked resources");

        driver.quit();
    }
}

// ─── CODE EXAMPLE 4: Full Page Screenshot (Firefox) — New in Selenium 4 ───
// Standard getScreenshotAs() only captures visible viewport
// Firefox in Selenium 4 supports full scrollable page capture

import org.openqa.selenium.OutputType;
import org.openqa.selenium.firefox.FirefoxDriver;
import org.openqa.selenium.firefox.HasFullPageScreenshot; // Firefox-only interface
import org.apache.commons.io.FileUtils;
import java.io.File;

public class FullPageScreenshot {
    public static void main(String[] args) throws Exception {

        // Must use FirefoxDriver — this feature is Firefox-only for now
        FirefoxDriver driver = new FirefoxDriver();
        driver.get("https://www.selenium.dev/documentation/");

        // HasFullPageScreenshot → Firefox-specific interface for full-page capture
        // Cast driver to this interface to access getFullPageScreenshotAs()
        File fullPage = ((HasFullPageScreenshot) driver)
            .getFullPageScreenshotAs(OutputType.FILE);
        // This captures ENTIRE page height — not just visible viewport

        // Save the screenshot to disk
        FileUtils.copyFile(fullPage, new File("full-page-screenshot.png"));

        // Regular screenshot for comparison (viewport only)
        File viewport = driver.getScreenshotAs(OutputType.FILE);
        FileUtils.copyFile(viewport, new File("viewport-screenshot.png"));

        driver.quit();
    }
}

// ─── CODE EXAMPLE 5: WebDriver BiDi — Real-time Event Listening ───
// BiDi = Bidirectional WebDriver Protocol (W3C draft standard)
// Enables PUSH events from browser to Java (vs old pull/polling model)

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.chrome.ChromeOptions;
import org.openqa.selenium.bidi.browsingcontext.BrowsingContext;
import org.openqa.selenium.bidi.module.LogInspector;

public class BiDiDemo {
    public static void main(String[] args) {

        // Enable BiDi by setting "webSocketUrl" capability to true
        ChromeOptions options = new ChromeOptions();
        options.setCapability("webSocketUrl", true); // activates BiDi mode

        WebDriver driver = new ChromeDriver(options);

        // LogInspector → BiDi module that listens for browser log events
        // This is a PUSH model — events fire as they happen, no polling
        try (LogInspector inspector = new LogInspector(driver)) {

            // onConsoleEntry() → fires on EVERY console.log/error/warn call
            inspector.onConsoleEntry(entry ->
                System.out.println("BiDi Console: " + entry.getText())
            );

            driver.get("https://www.example.com");
            System.out.println("BiDi event listener active!");
        }

        driver.quit();
    }
}

✅ Best Practices

  • Use CDP for Chrome-specific advanced scenarios only — keep core test logic browser-agnostic using standard WebDriver
  • Use HasFullPageScreenshot in Firefox for visual regression testing of long scrollable pages
  • Use BiDi protocol instead of CDP when cross-browser compatibility matters (BiDi is a W3C standard, CDP is Chrome-only)
  • Prefer Selenium Grid 4 with Docker Compose for consistent, reproducible local Grid environments
  • Prefix CDP DevTools version in import paths (e.g., v120) — update when Chrome major versions change
  • Monitor the Grid dashboard at localhost:4444 in Grid 4 to observe active sessions and node health

🎯 SDET Interview Q&A

Q1. What new features does Selenium 4 offer for network testing?
A: Via Chrome DevTools Protocol (CDP): intercept and block network requests, mock responses, monitor all outgoing HTTP calls, and simulate offline mode — all from Java code without proxy tools like BrowserMob.
Q2. What is WebDriver BiDi and how is it different from CDP?
A: BiDi (Bidirectional) is a W3C draft standard for real-time browser-to-driver communication. CDP is Chrome-proprietary. BiDi works across Chrome, Firefox, and Edge. It enables push-based event listening rather than polling.
Q3. Can you take a full-page screenshot in Selenium 4?
A: Yes, but only in Firefox using the HasFullPageScreenshot interface and getFullPageScreenshotAs() method. Chrome's standard getScreenshotAs() captures only the visible viewport unless you use CDP scroll-capture workarounds.
Q4. How is Selenium Grid 4 different from Grid 3?
A: Grid 4 is completely rewritten. New components: Router, Distributor, SessionMap, SessionQueue, Node. New features: native Docker/Kubernetes support, built-in GraphQL monitoring API, UI dashboard at port 4444, and Standalone mode (single JAR for Hub+Node).
Q5. How do you mock geolocation in Selenium 4?
A: Use Chrome DevTools Protocol's Emulation domain — devTools.send(Emulation.setGeolocationOverride(lat, lng, accuracy)). The browser will then report those coordinates to any website requesting GPS location via the Geolocation API.
