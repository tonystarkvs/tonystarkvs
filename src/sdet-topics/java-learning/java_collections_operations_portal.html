LinkedList implements both List and Deque. Every node is an object with data + prev/next pointers. Optimal for frequent head/tail insertions, queue/stack use. Avoid index-based access — it's O(n).

OrderedDuplicates ✓Null ✓ head/tail O(1)get(index) O(n)Not Thread-Safe

new LinkedList<>() / from collection / as Queue / as Deque / as StackEmpty or populated▼
import java.util.*;

// 1. Empty LinkedList as List
List<String> ll = new LinkedList<>();

// 2. From existing collection
List<Integer> ll2 = new LinkedList<>(List.of(1,2,3));

// 3. As Queue (FIFO)
Queue<String> q = new LinkedList<>();

// 4. As Deque (double-ended queue)
Deque<String> dq = new LinkedList<>();

// 5. As Stack (LIFO) via Deque
Deque<String> stack = new LinkedList<>();
💡 Prefer ArrayDeque for pure stack/queue use — better cache performance. Use LinkedList when you need both List and Deque features.
② ADD OPERATIONS 10 ways
add / addFirst / addLast / offer / offerFirst / offerLast / push / addAll / add(index)boolean / void▼
LinkedList<String> ll = new LinkedList<>();

// 1. add(E) — appends to TAIL — O(1)
ll.add("A"); ll.add("B"); ll.add("C");

// 2. addFirst(E) — prepend to HEAD — O(1)
ll.addFirst("HEAD"); // [HEAD, A, B, C]

// 3. addLast(E) — append to TAIL — O(1) same as add()
ll.addLast("TAIL"); // [HEAD, A, B, C, TAIL]

// 4. add(int index, E) — insert at position — O(n) traversal
ll.add(2, "X"); // [HEAD, A, X, B, C, TAIL]

// 5. offer(E) — Queue API: adds to tail, returns true/false
ll.offer("Q1");

// 6. offerFirst(E) — Deque: add to head
ll.offerFirst("QF");

// 7. offerLast(E) — Deque: add to tail
ll.offerLast("QL");

// 8. push(E) — Stack: pushes to HEAD (same as addFirst)
ll.push("STACK-TOP");

// 9. addAll(Collection) — bulk add to tail
ll.addAll(List.of("P","Q","R"));

// 10. addAll(int index, Collection) — bulk insert at position
ll.addAll(0, List.of("START1","START2"));
💡 addFirst/addLast, offerFirst/offerLast, push/offer all ultimately modify head or tail pointers in O(1) — they are just different API names from List, Deque, and Queue interfaces.
③ UPDATE OPERATIONS 3 ways
set(index) / replaceAll / listIterator.setE (old value)▼

// 2. replaceAll — transform every element (Java 8)

// 3. ListIterator.set — most efficient for sequential update

    String s = lit.next();

⚠️ Avoid set(index) in large LinkedLists — it traverses from head or tail to the index each time. Use ListIterator for sequential updates.
④ DELETE OPERATIONS 10 ways
remove / removeFirst / removeLast / poll / pollFirst / pollLast / pop / removeIf / clear / removeAllE / boolean▼

// 2. remove(int index) — remove by index — O(n)

// 3. remove(Object) — remove first occurrence — O(n)

TreeMap<Integer,String> tm = new TreeMap<>(
    Map.of(10,"A",20,"B",30,"C",40,"D",50,"E"));

// 1. firstKey / lastKey
System.out.println(tm.firstKey()); // 10
System.out.println(tm.lastKey());  // 50

// 2. firstEntry / lastEntry (key+value)
System.out.println(tm.firstEntry()); // 10=A
System.out.println(tm.lastEntry());  // 50=E

// 3. floorKey — greatest key ≤ given
System.out.println(tm.floorKey(25));   // 20
System.out.println(tm.floorEntry(25)); // 20=B

// 4. ceilingKey — smallest key ≥ given
System.out.println(tm.ceilingKey(25));   // 30
System.out.println(tm.ceilingEntry(25)); // 30=C

// 5. lowerKey — strictly less than given
System.out.println(tm.lowerKey(30)); // 20

// 6. higherKey — strictly greater than given
System.out.println(tm.higherKey(30)); // 40

// 7. headMap(toKey) — keys STRICTLY less than toKey
System.out.println(tm.headMap(30));        // {10=A, 20=B}
System.out.println(tm.headMap(30,true));   // {10=A, 20=B, 30=C}

// 8. tailMap(fromKey) — keys ≥ fromKey
System.out.println(tm.tailMap(30));        // {30=C, 40=D, 50=E}
System.out.println(tm.tailMap(30,false));  // {40=D, 50=E}

// 9. subMap(from, to) — [from, to)
System.out.println(tm.subMap(20,40));              // {20=B, 30=C}
System.out.println(tm.subMap(20,true,40,true));   // {20,30,40}

// 10. descendingMap — reverse-sorted view
System.out.println(tm.descendingMap()); // {50,40,30,20,10}

// 11. descendingKeySet
System.out.println(tm.descendingKeySet()); // [50,40,30,20,10]

// 12. containsKey / containsValue
System.out.println(tm.containsKey(20));    // true — O(log n)
System.out.println(tm.containsValue("C")); // true — O(n)
⑥ ITERATION 7 ways
entrySet / keySet / values / forEach / stream / descendingMap / headMap/tailMap iterationvoid — always in sorted key order▼
TreeMap<Integer,String> tm = new TreeMap<>(
    Map.of(30,"C",10,"A",50,"E",20,"B",40,"D"));

// 1. entrySet for-each — ascending key order
for (Map.Entry<Integer,String> e : tm.entrySet())
    System.out.println(e.getKey() + " → " + e.getValue());
// 10→A, 20→B, 30→C, 40→D, 50→E

// 2. keySet for-each — ascending
for (int k : tm.keySet()) System.out.print(k + " ");

// 3. values only — in key-sorted order
for (String v : tm.values()) System.out.print(v + " ");

// 4. forEach lambda
tm.forEach((k, v) -> System.out.println(k + ":" + v));

// 5. Stream — already sorted
tm.entrySet().stream()
    .filter(e -> e.getKey() > 20)
    .forEach(e -> System.out.println(e.getKey() + "=" + e.getValue()));

// 6. Descending iteration via descendingMap
tm.descendingMap().forEach((k,v) -> System.out.println(k+"="+v));
// 50→E, 40→D, 30→C, 20→B, 10→A

// 7. Range iteration — only keys in [20, 40]
tm.subMap(20, true, 40, true)
  .forEach((k,v) -> System.out.println(k+"="+v));
// 20=B, 30=C, 40=D
💡 TreeMap iteration is ALWAYS in sorted key order — no sort() call needed. Use descendingMap() or descendingKeySet() for reverse order. Use subMap/headMap/tailMap for efficient range iteration without scanning the whole map.
