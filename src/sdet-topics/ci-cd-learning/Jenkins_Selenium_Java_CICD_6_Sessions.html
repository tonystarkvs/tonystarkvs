FOUNDATION — Jenkins + CI/CD + Selenium Java Basics

Understand Jenkins and how Selenium Java + TestNG + Maven tests run inside a CI/CD pipeline. (Beginner • Session 1 of 6)

Overview — What / Why / When / Where / How

What Jenkins is an automation server. Selenium Java is a browser automation library used with TestNG and Maven to build UI test frameworks.
Why It runs Selenium tests automatically after every push so we catch UI bugs early, not during release week.
When When the team wants smoke checks on every commit and scheduled regression at night.
Where On a Jenkins server or agent that has Java, Maven, Git, and Chrome ready.
How Developer pushes code ➜ Jenkins triggers job ➜ Maven compiles ➜ TestNG runs Selenium tests ➜ reports archived.

CI
Continuous Integration with frequent merges and automated checks.
CD
Continuous Delivery or Deployment of builds to environments.
Pipeline
Sequence of steps like checkout, build, test, report.
Selenium
Library to drive Chrome, Firefox, Edge browsers.

Java test framework for grouping, ordering, parallel runs.
Maven
Java build tool for dependencies and lifecycle.
WebDriver
Selenium interface that controls a browser session.
Selenium Manager
Built-in Selenium 4 driver resolver — no more WebDriverManager.
Test Pack
Group of tests like smoke, sanity, regression.
Artifact
Saved output like reports, screenshots, surefire XML.

Interview One-Liners

Jenkins is the runner; Selenium Java is the browser test engine.
Every push should answer one question — is the app still working?
A good Selenium framework keeps smoke tests fast and stable.
Selenium Manager removed the old WebDriverManager pain in Selenium 4.
Reports and screenshots make Jenkins failures understandable to the team.

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.chrome.ChromeOptions;
import org.testng.annotations.AfterMethod;
import org.testng.annotations.BeforeMethod;

public class BaseTest {
    protected WebDriver driver;

    @BeforeMethod
    public void setUp() {
        // Selenium Manager auto-downloads the right driver
        ChromeOptions options = new ChromeOptions();
        options.addArguments("--start-maximized");
        driver = new ChromeDriver(options);
    }

    @AfterMethod
    public void tearDown() {
        if (driver != null) driver.quit();
    }
}

Best Practices

  • ✓Start with a small stable smoke pack before adding regression.
  • ✓Use Maven Surefire to run TestNG — keep it standard.
  • ✓Let Selenium Manager handle drivers — do not commit chromedriver.exe.
  • ✓Use explicit waits with Duration.ofSeconds, never Thread.sleep.
  • ✓Name jobs like SE-Java-Smoke-QA or SE-Regression-Nightly.
  • ✓Store test code in Git; never edit scripts on the Jenkins server.
  • ✓Always archive Surefire reports, Extent reports, and screenshots.
  • ✓Externalize URLs and credentials into config files or Jenkins credentials.
  • ✓Keep the smoke pack under a few minutes for quick feedback.
  • ✓Document failure ownership: app, test, environment, or data.

Interview Prep — Q&A

Q1. What is Jenkins in simple words?
A: Jenkins is a tool that automatically runs build and test steps without manual effort.
Q2. Why use Selenium with Jenkins?
A: Because Jenkins can trigger Selenium tests on every code change and share results with the team.
Q3. What is Selenium Manager?
A: A Selenium 4 feature that auto-downloads the correct browser driver.
Q4. What is a TestNG suite?
A: An XML file that decides which tests run, with which parameters and groups.
Q5. Smoke vs regression in one line?
A: Smoke is quick critical checks; regression is broad coverage that runs less often.

● Jenkinsfile 
pipeline {
  agent any
  tools { jdk 'JDK17'; maven 'Maven3' }

  parameters {
    choice(name: 'ENV', choices: ['qa', 'uat'], description: 'Target environment')
    choice(name: 'BROWSER', choices: ['chrome', 'firefox', 'edge'], description: 'Browser')
    choice(name: 'SUITE', choices: ['testng-smoke.xml', 'testng-regression.xml'], description: 'Suite file')
  }

  environment {
    QA_URL  = 'https://qa.example.com'
    UAT_URL = 'https://uat.example.com'
  }

  stages {
    stage('Checkout') { steps { checkout scm } }

    stage('Run Selected Suite') {
      steps {
        script {
          def url = params.ENV == 'qa' ? env.QA_URL : env.UAT_URL
          sh "BASE_URL=${url} BROWSER=${params.BROWSER} mvn -B clean test -DsuiteXmlFile=${params.SUITE}"
        }
      }
    }
  }

  post {
    always {
      archiveArtifacts artifacts: 'target/surefire-reports/**, reports/**', allowEmptyArchive: true
    }
  }
}

package com.learn.utils;

import java.io.FileInputStream;
import java.util.Properties;

public class ConfigReader {
    private static final Properties props = new Properties();

    static {
        try (FileInputStream fis = new FileInputStream("src/test/resources/config.properties")) {
            props.load(fis);
        } catch (Exception e) {
            throw new RuntimeException("Could not load config", e);
        }
    }

    public static String get(String key) {
        // Jenkins env overrides config file
        String envKey = key.toUpperCase().replace('.', '_');
        return System.getenv().getOrDefault(envKey, props.getProperty(key));
    }
}

package com.learn.utils;

import org.apache.poi.ss.usermodel.*;
import org.apache.poi.xssf.usermodel.XSSFWorkbook;
import java.io.FileInputStream;
import java.util.*;

public class ExcelReader {
    public static List<Map<String, String>> readSheet(String path, String sheetName) throws Exception {
        List<Map<String, String>> rows = new ArrayList<>();
        try (Workbook wb = new XSSFWorkbook(new FileInputStream(path))) {
            Sheet sheet = wb.getSheet(sheetName);
            Row header = sheet.getRow(0);
            for (int r = 1; r <= sheet.getLastRowNum(); r++) {
                Row row = sheet.getRow(r);
                Map<String, String> map = new LinkedHashMap<>();
                for (int c = 0; c < header.getLastCellNum(); c++) {
                    map.put(header.getCell(c).getStringCellValue(), row.getCell(c).toString());
                }
                rows.add(map);
            }
        }
        return rows;
    }
}

Best Practices

  • ✓Externalize test data into Excel, JSON, or properties.
  • ✓Use unique users/orders for flows that modify records.
  • ✓Reset data after destructive test cases.
  • ✓Start parallel=methods with small thread-count, then scale.

● src/test/java/com/learn/listeners/ScreenshotListener.java 
package com.learn.listeners;

import org.openqa.selenium.*;
import org.openqa.selenium.io.FileHandler;
import org.testng.ITestListener;
import org.testng.ITestResult;
import java.io.File;

public class ScreenshotListener implements ITestListener {
    @Override
    public void onTestFailure(ITestResult result) {
        Object current = result.getInstance();
        try {
            WebDriver driver = (WebDriver) current.getClass()
                    .getDeclaredField("driver").get(current);
            File src = ((TakesScreenshot) driver).getScreenshotAs(OutputType.FILE);
            String name = result.getName() + "-" + System.currentTimeMillis() + ".png";
            FileHandler.copy(src, new File("reports/screenshots/" + name));
        } catch (Exception e) {
            // Log silently for now
        }
    }
}
