S21 — Python CLI Commands
Every essential command-line command for Python: running scripts · pip · venv · debugging · interactive shell
Phase 1 — Fundamentals
Overview — What / Why / When / How
What is Python CLI?
The Command Line Interface lets you run Python scripts, manage packages, create virtual environments, and debug — all without any IDE. It is the backbone of every QA automation pipeline and CI/CD system.
Why master CLI?
Servers, CI runners (GitHub Actions, Jenkins), and Docker containers have no GUI. Every SDET must control Python entirely from the terminal to install dependencies, run tests, and read output.

Interview prep Q&A
Q1: What is the difference between python and python3 on the command line? +
On Windows, python usually maps to the latest installed version. On macOS/Linux, python may still point to Python 2.x (legacy). python3 always refers to Python 3.x. In modern setups (Python 3.11+) and on Windows with the py launcher, python and python3 behave identically.
Q2: How do you run only smoke-tagged tests from the CLI? +
python -m pytest -m smoke tests/ — the -m flag filters by marker. Combined: pytest -m "smoke and not slow" -n auto --html=report.html. Markers must be registered in pytest.ini.
Q3: What does pip install -e . do? +
Installs the current directory package in "editable" mode. Changes to source files are reflected immediately without re-installing. Used in framework development so you can modify helper modules and re-run tests without reinstalling.
Q4: How do you pass environment variables to a Python script from CLI? +
On Linux/Mac: TEST_ENV=staging python run.py. On Windows: set TEST_ENV=staging && python run.py. Inside the script: os.getenv("TEST_ENV"). In CI, set them as pipeline secrets or environment variables in the workflow YAML.
Q5: What is sys.argv and how is it used in QA? +
sys.argv is a list where argv[0] is the script name and argv[1:] are command-line arguments. QA use: python run.py staging chrome — inside the script, sys.argv[1] gives "staging" and sys.argv[2] gives "chrome" to configure the test run dynamically.

1 — Python 3.6: f-strings (PEP 498)3.6+
# Before 3.6 — verbose .format()
name, score = "Alice", 98.5
old = "Tester: {}, Score: {:.1f}".format(name, score)

# Python 3.6+ — f-strings (faster + cleaner)
modern = f"Tester: {name}, Score: {score:.1f}"

# Expressions inside f-strings
items = ["TC1", "TC2", "TC3"]
print(f"Total tests: {len(items)}")
print(f"Pass rate: {98/100:.0%}")   # 98%

# 3.12: f-strings allow nested quotes (backslash OK)
print(f"First: {items[0]!r}")       # 'TC1'
# Output: Total tests: 3 | Pass rate: 98%
2 — Python 3.7: dataclasses (PEP 557)3.7+
from dataclasses import dataclass, field

@dataclass
class TestCase:
    tc_id:    str
    title:    str
    priority: str  = "Medium"
    tags:     list = field(default_factory=list)
    passed:   bool = False

tc = TestCase("TC001", "Login test", priority="High")
print(tc)
# TestCase(tc_id='TC001', title='Login test', priority='High'...)

# Auto-generates __init__, __repr__, __eq__ — no boilerplate!
tc2 = TestCase("TC001", "Login test", priority="High")
print(tc == tc2)   # True — __eq__ is auto-generated
3 — Python 3.8: walrus operator := (PEP 572)3.8+
# Without walrus — two lines
data = [10, 25, 3, 87, 5]
filtered = []
for x in data:
    result = x * 2
    if result > 30:
        filtered.append(result)

# With walrus := — assign AND test in one expression
filtered = [y for x in data if (y := x * 2) > 30]
print(filtered)     # [50, 174, 10] — wait, 87*2=174

# QA use: read response AND check in one step
import requests
if (n := len(requests.get("https://jsonplaceholder.typicode.com/users").json())) > 5:
    print(f"Got {n} users")
4 — Python 3.10: match-case (PEP 634)3.10+
# match-case = powerful switch statement
def describe_status(code):
    match code:
        case 200:
            return "OK"
        case 201:
            return "Created"
        case 401 | 403:     # OR pattern
            return "Auth error"
        case code if code >= 500:  # guard clause
            return f"Server error {code}"
        case _:               # default / wildcard
            return "Unknown"

print(describe_status(403))   # Auth error
print(describe_status(503))   # Server error 503
5 — Python 3.9 & 3.10: type hints + union
# 3.9: built-in generics (no import needed)
def get_test_ids(suite: str) -> list[str]:   # list[str] not List[str]
    return ["TC001", "TC002"]

# 3.9: dict merge operator |
defaults = {"timeout": 30, "retries": 3}
overrides = {"timeout": 60, "env": "staging"}
config = defaults | overrides
print(config)   # {'timeout': 60, 'retries': 3, 'env': 'staging'}

# 3.10: X | Y union type (replaces Optional[X])
def find_user(uid: int) -> dict | None:
    return {"id": uid} if uid > 0 else None

Interview prep Q&A
Q1: What broke when moving from Python 2 to 3? +
Major breaks: print became a function, integer division changed (5/2=2.5 not 2), unicode became the default string type, range() returns an iterator not a list, and many standard library modules were renamed. The 2to3 tool automates most migration.
Q2: What Python version should you target for new SDET projects? +
Python 3.11 or 3.12. They offer the best performance (3.11 is ~60% faster than 3.10 on benchmarks), best type hint support, and all modern features. Always check that your key dependencies (selenium, pytest, requests) support the target version first.
Q3: What is PEP and why does it matter? +
PEP = Python Enhancement Proposal. It's how new language features are proposed, discussed, and approved. Key PEPs: PEP 8 (style guide), PEP 498 (f-strings), PEP 572 (walrus), PEP 634 (match-case). Knowing a feature's PEP number in interviews signals deep familiarity.
Q4: How do you check Python version programmatically? +
import sys; print(sys.version_info) — returns a named tuple: sys.version_info(major=3, minor=11, micro=4). Use sys.version_info >= (3, 10) in guards to conditionally enable version-specific features.
Q5: What is the dict merge operator | introduced in 3.9? +
c = a | b merges two dicts (b wins on conflicts). a |= b merges in place. Before 3.9, you had to use {**a, **b} or a.update(b). Used in QA to merge default configs with environment-specific overrides.

S23 — Reusable Functions, Multiple Returns & **kwargs
Writing clean, reusable functions · multiple return patterns · deep-dive into *args and **kwargs with 10+ examples
Phase 2 — Intermediate
Overview
What are reusable functions?
Functions that accept flexible inputs, return meaningful values, and can be called from any part of your test framework without modification. The cornerstone of DRY (Don't Repeat Yourself) automation code.
What is **kwargs?
**kwargs (keyword arguments) collects any number of named arguments into a dictionary. It lets functions be flexible — callers can pass any key=value pairs without changing the function signature.
10 code examples
1 — Basic reusable function
def build_url(base, endpoint, version="v1"):
    """Build a full API URL. Reusable across all tests."""
    return f"{base}/{version}/{endpoint}"

print(build_url("https://qa.example.com", "users"))
# https://qa.example.com/v1/users
print(build_url("https://qa.example.com", "orders", "v2"))
# https://qa.example.com/v2/orders
2 — Multiple return values
def parse_response(resp):
    """Return status, body, and time as a tuple."""
    return resp.status_code, resp.json(), resp.elapsed.total_seconds()

# Unpack directly — clear and readable

def run_test(tc_id, fn):

def create_user(**kwargs):

Interview prep Q&A
Q1: What is the correct order of parameters in a function signature? +
Strict order: def func(regular, /, positional_only, *, keyword_only). In practice: def func(pos_args, *args, default=val, **kwargs). Positional always before *args, defaults before **kwargs. Mixing out of order raises SyntaxError.
Q2: What is the mutable default argument bug? +
def f(x=[]) is dangerous — the list is created once and shared across all calls. f(); f() both append to the same list. Fix: def f(x=None): x = x or []. Or use dataclasses.field(default_factory=list).
Q3: How do you enforce keyword-only arguments? +
Place a bare * in the signature: def login(url, *, username, password). Everything after * must be passed as keyword — callers cannot use positional for username or password. Prevents argument order bugs.
Q4: What does functools.wraps do in a wrapper function? +
It copies the wrapped function's __name__, __doc__, and other attributes onto the wrapper. Without it, func.__name__ returns "wrapper" — making logs, test names, and stack traces unreadable.
Q5: What is the difference between return None and no return statement? +
Functionally identical — Python implicitly returns None at the end of every function. But explicitly writing return None is a signal to readers that the function intentionally returns nothing (vs. returning a value). In QA helpers, prefer explicit return for clarity.

S24 — Lambda Functions (Anonymous One-Liners)
Syntax · use cases · map() · filter() · sorted() · reduce() · callbacks · 10 real-world examples
Phase 2 — Intermediate
Overview
What is a lambda?
An anonymous, single-expression function defined inline without a name. Syntax: lambda params: expression. Returns the expression result automatically — no return keyword needed.
When to use?
Use lambdas for short, throwaway functions passed as arguments to sorted(), map(), filter(), or event callbacks. Avoid for complex logic — use a named def instead.
10 examples
1 — Basic lambda
# Named function vs lambda equivalent
def square(x): return x ** 2
square_l = lambda x: x ** 2

print(square(5))    # 25
print(square_l(5))  # 25

# QA: quick status checker
is_success = lambda code: code in (200, 201, 204)
print(is_success(201))  # True
print(is_success(500))  # False
2 — sorted() with lambda key
tests = [
    {"id": "TC3", "priority": 3},
    {"id": "TC1", "priority": 1},
    {"id": "TC2", "priority": 2},
]
# Sort by priority ascending
ordered = sorted(tests, key=lambda t: t["priority"])
print([t["id"] for t in ordered])
# ['TC1', 'TC2', 'TC3']

# Sort descending (reverse)
hi_first = sorted(tests, key=lambda t: t["priority"], reverse=True)
3 — filter() with lambda
results = [
    ("TC1", "PASS"), ("TC2", "FAIL"),
    ("TC3", "PASS"), ("TC4", "SKIP"),
]
# Keep only failures
failures = list(filter(lambda r: r[1] == "FAIL", results))
print(failures)
# [('TC2', 'FAIL')]

# Equivalent list comprehension (preferred in modern Python)
failures2 = [r for r in results if r[1] == "FAIL"]
4 — map() with lambda
endpoints = ["users", "orders", "products"]
base = "https://qa.example.com/api/v1"

# map applies lambda to every item
full_urls = list(map(lambda e: f"{base}/{e}", endpoints))
print(full_urls)
# ['https://qa.example.com/api/v1/users', ...]

# Normalize test IDs
raw = ["tc1", " TC2 ", "tc3"]
clean = list(map(lambda x: x.strip().upper(), raw))
# ['TC1', 'TC2', 'TC3']
5 — reduce() with lambda
from functools import reduce

durations = [1.2, 0.8, 2.1, 0.5]

# Sum all durations
total = reduce(lambda acc, x: acc + x, durations)
print(f"Total: {total:.1f}s")   # Total: 4.6s

# Find max duration
slowest = reduce(lambda a, b: a if a > b else b, durations)
print(f"Slowest: {slowest}s")  # Slowest: 2.1s
6 — Multi-param lambda
# Lambda with multiple parameters
add = lambda x, y: x + y
print(add(3, 7))    # 10

# Format test result line
fmt_result = lambda tc, st, dur: f"[{tc}] {st} ({dur:.2f}s)"
print(fmt_result("TC001", "PASS", 1.35))
# [TC001] PASS (1.35s)
7 — Lambda with conditional (ternary)
# Ternary expression inside lambda
grade = lambda score: "PASS" if score >= 90 else "FAIL"
print(grade(95))   # PASS
print(grade(75))   # FAIL

# Apply to a list
scores = [92, 78, 95, 60]
grades = list(map(grade, scores))
# ['PASS', 'FAIL', 'PASS', 'FAIL']

9 — sorted() with multi-key lambda
tests = [
    ("TC1", "High", 2.1),
    ("TC2", "Low",  0.5),
    ("TC3", "High", 0.8),
    ("TC4", "Low",  1.1),
]
# Sort by priority then by duration

10 — Lambda limitation: when NOT to use
# ❌ BAD: lambda is too complex — unreadable
process = lambda r: (None if r.status_code != 200
                     else r.json().get("user", {}).get("id"))

# ✅ GOOD: use a named function instead
def get_user_id(response):
    """Extract user ID from API response."""
    if response.status_code != 200:
        return None
    return response.json().get("user", {}).get("id")
# Rule: if you need a docstring, use def not lambda
Interview one-liners
  • 1"Lambda is an anonymous function for simple, throwaway operations — it has no name, no docstring, and can only contain a single expression."
  • 2"sorted(list, key=lambda x: x['priority']) is the idiomatic way to sort complex objects by a specific field in Python."
  • 3"filter(lambda x: x > 0, items) returns an iterator of items satisfying the condition — equivalent to a list comprehension with an if clause."
  • 4"reduce(lambda acc, x: acc + x, items) folds a list into a single value by applying the function cumulatively — useful for totalling test durations."
  • 5"If a lambda needs more than one expression or requires a docstring, it should be a named def function — readability always wins."
Interview prep Q&A
Q1: What is the difference between lambda and def? +
Lambda creates an anonymous function inline with a single expression — no statements, no return keyword, no docstring. def creates a named function with full body, multiple statements, and docstring support. PEP 8 discourages assigning lambdas to variables — use def instead.
Q2: Can a lambda have multiple lines? +
No. Lambda is restricted to a single expression. You cannot use if/else statements (only ternary), loops, try/except, or assignments. These restrictions are intentional — they keep lambdas small and force complex logic into named functions.
Q3: What does map() return in Python 3? +
A lazy iterator, not a list. map(func, items) only computes values when iterated. Wrap with list() to materialise. This is memory-efficient for large datasets but can surprise you if you expect a list immediately.
Q4: When would you choose filter() over list comprehension? +
Rarely — list comprehensions are more readable and Pythonic for most cases. filter() is useful when passing a predicate function as a parameter, or when composing functional pipelines. [x for x in items if x > 0] is clearer than list(filter(lambda x: x > 0, items)).
Q5: What is a closure and how does it relate to lambda? +
A closure is a function that captures variables from its enclosing scope. Lambda functions can be closures too: def multiplier(n): return lambda x: x * n — the returned lambda captures n. Used in QA to create parameterised validator functions dynamically.

Edge cases and common mistakes
# ❌ IndexError: list index out of range
lst = [1, 2, 3]
# lst[5]  ← IndexError!

# ✅ Safe access with try/except or get
try:
    val = lst[10]
except IndexError:
    val = None

# ❌ Modifying list WHILE iterating — skips items!
nums = [1, 2, 3, 4]
for n in nums:       # WRONG — unpredictable!
    if n % 2 == 0: nums.remove(n)

# ✅ Iterate over a copy
for n in nums[:]:    # iterate copy, modify original
    if n % 2 == 0: nums.remove(n)

  • 2"append() adds one item to the end; extend() adds all items from an iterable — confusing them causes nested lists."
  • 3"remove() deletes the first matching value; pop() deletes by index and returns the item — use pop when you need the removed value."

Interview prep Q&A
Q1: Difference between append() and extend()? +
append(x) adds x as a single item — [1,2].append([3,4]) → [1,2,[3,4]]. extend(iterable) adds all items individually — [1,2].extend([3,4]) → [1,2,3,4]. A common SDET bug is using append when extend was intended, creating nested lists.
Q2: How is a list different from a tuple? +
List = mutable (items can change), tuple = immutable (fixed after creation). Lists use more memory and are slightly slower for iteration. Tuples are appropriate for fixed test data (status code constants, browser configurations) that should not be accidentally modified.
Q3: What does list.sort() return? +
None — it sorts in place and explicitly returns None. A common mistake: result = my_list.sort() assigns None. Use sorted(my_list) when you need a new sorted list without modifying the original.
Q4: How do you flatten a nested list? +
[item for sub in nested for item in sub] — a double comprehension. Or list(itertools.chain.from_iterable(nested)). For deeply nested: use recursion or itertools.chain with unpacking.
Q5: What is list slicing and what does list[:] do? +
Slicing creates a new list from a range: lst[start:stop:step]. lst[:] creates a shallow copy of the entire list — often used in QA to safely iterate over a list while modifying the original during a loop.

S26 — Tuple Collection: Complete Operations Guide
Immutable sequences · packing/unpacking · indexing · why modify is blocked · single-element pitfall
Phase 2 — Collections
What is a tuple?
An ordered, immutable sequence. Once created it cannot be modified. Use tuples for constant data: HTTP status codes, browser configs, function return pairs — anything that must not change.
Why immutable?
Immutability = safety + speed. Tuples are hashable (can be dict keys / set members). Slightly faster iteration than lists. They signal to readers: "this data is not meant to change."
Creating tuples
# Empty tuple
empty = ()
empty2 = tuple()

# With values
browsers = ("chrome", "firefox", "edge")
point    = (10, 20)

# ⚠️ Single-element MUST have trailing comma!
single_correct = ("chrome",)   # tuple  ← correct
not_a_tuple    = ("chrome")    # str   ← just parentheses!
print(type(single_correct))   # <class 'tuple'>
print(type(not_a_tuple))      # <class 'str'>

# From list
lst = [1, 2, 3]
t   = tuple(lst)

# Tuple packing (no parentheses needed)
status = 200, "OK", 1.2   # automatically packed as tuple
Accessing and looping
codes = (200, 201, 204, 400, 404, 500)

print(codes[0])     # 200   (same indexing as list)
print(codes[-1])    # 500
print(codes[1:4])    # (201, 204, 400)

# Check membership
print(404 in codes)  # True

# Loop
for code in codes:
    print(code, end=" ")
# 200 201 204 400 404 500

# count() and index()
t = (1, 2, 2, 3)
print(t.count(2))    # 2
print(t.index(3))    # 3
Tuple unpacking (powerful Python feature)
# Basic unpacking
host, port = ("localhost", 5432)
print(host)   # localhost
print(port)   # 5432

# Swap variables elegantly
a, b = 10, 20
a, b = b, a
print(a, b)   # 20 10

# Extended unpacking with *
first, *rest = (1, 2, 3, 4, 5)
print(first)  # 1
print(rest)   # [2, 3, 4, 5]

# Unpack function return (S23 pattern)
def get_pair(): return "Alice", "admin"
name, role = get_pair()   # clean unpacking
Immutability: modify workaround
t = ("chrome", "firefox", "edge")

# ❌ Tuples don't support item assignment
# t[0] = "safari"  ← TypeError: 'tuple' object does not support item assignment

# ✅ Workaround: convert → modify → convert back
lst = list(t)       # tuple → list
lst[0] = "safari"
t = tuple(lst)     # list → tuple
print(t)            # ('safari', 'firefox', 'edge')

# ✅ Concatenate to "extend" a tuple
t2 = t + ("opera",)
print(t2)
Edge cases
# ❌ Pitfall 1: single element without comma
wrong = (42)         # int, not tuple!
right = (42,)        # tuple with one int
print(type(wrong))   # <class 'int'>
print(type(right))   # <class 'tuple'>

# ❌ Pitfall 2: del individual element not allowed
t = (1, 2, 3)
# del t[0]  ← TypeError!

# ✅ del entire tuple is allowed
del t
# print(t)  ← NameError: t is not defined

# Tuple as dict key (because it's hashable)
cache = {("GET", "/users"): "cached_response"}
print(cache[("GET", "/users")])

  • 1"Tuples are immutable sequences — unlike lists, their items cannot be changed after creation, making them safe for constant configuration data."
  • 2"A single-element tuple requires a trailing comma — (42,) is a tuple, but (42) is just parentheses around an integer."
  • 3"Tuple unpacking lets you assign multiple values in one line — a, b = my_tuple — and is the Pythonic way to consume multiple return values."
  • 4"Tuples are hashable, so they can be used as dictionary keys or set members — lists cannot because they are mutable."
  • 5"Named tuples (from collections.namedtuple) give field names to tuple positions — providing dict-like readability with tuple performance."
Interview prep Q&A
Q1: Why use a tuple over a list for HTTP status codes? +
Tuples signal immutability — the codes won't accidentally change. They are slightly faster to iterate, hashable (can be set members), and communicate intent: this is a fixed constant, not a dynamic collection. VALID = (200, 201, 204) is safer than a list.
Q2: Can a tuple contain mutable objects? +
Yes — a tuple's contents are fixed, but if a contained object is mutable (like a list), that object can still be modified. Example: t = ([1,2], [3,4]); t[0].append(9) is valid. The tuple reference is fixed, but the list inside can change.
Q3: What is tuple packing and unpacking? +
Packing: assigning multiple values to one variable without explicit parentheses — t = 1, 2, 3. Unpacking: assigning tuple positions to multiple variables — a, b, c = t. Extended unpacking with * captures remaining items into a list.
Q4: What is a namedtuple? +
from collections import namedtuple; Point = namedtuple('Point', ['x','y']); p = Point(10,20); p.x — it gives names to tuple positions. Used in QA for test result records: Result = namedtuple('Result', ['id','status','duration']). Lightweight dataclass alternative.
Q5: How does tuple perform vs list for read-heavy operations? +
Tuples are stored more compactly and are slightly faster to create and iterate than lists. For configuration constants that are read thousands of times in a test suite (status codes, locators, env URLs), tuples are the better choice.

S27 — Set Collection: Complete Operations Guide
Unique, unordered items · set math (union, intersection, difference) · why indexing is unsupported
Phase 2 — Collections
What is a set?
An unordered collection of unique, hashable items. Backed by a hash table — O(1) membership testing. Perfect for deduplication, finding overlaps between test runs, and fast in-checks.
Why unordered?
Sets use hashing to store elements, not sequential slots. There is no "first" or "last" — no index. This is the trade-off for near-instant membership testing at any scale.
Creating sets
# Empty set — MUST use set(), not {} (that's a dict!)
empty = set()
print(type({}))          # <class 'dict'> — common mistake!
print(type(set()))       # <class 'set'>  ← correct

# With values (duplicates auto-removed)
tags = {"smoke", "login", "smoke", "critical"}
print(tags)               # {'smoke', 'login', 'critical'} — one 'smoke'

# From list (deduplicate)
raw   = ["TC1", "TC2", "TC1", "TC3"]
unique = set(raw)
print(unique)             # {'TC1', 'TC2', 'TC3'}
Accessing — membership check (no indexing)
browsers = {"chrome", "firefox", "edge"}

# Membership — O(1) — VERY fast even at scale
print("chrome" in browsers)    # True
print("safari" not in browsers) # True

# ❌ Indexing is NOT supported
# browsers[0]  ← TypeError: 'set' object is not subscriptable

# Loop (order is NOT guaranteed)
for b in browsers:
    print(b, end=" ")    # unpredictable order
Modifying sets
tags = {"smoke", "login"}

tags.add("critical")              # add one item
tags.update({"api", "regression"}) # add multiple
print(tags)
# {'smoke', 'login', 'critical', 'api', 'regression'}
Set math — the real power
run1_fails = {"TC1", "TC3", "TC5"}
run2_fails = {"TC3", "TC5", "TC7"}

# Union — all failures from both runs
print(run1_fails | run2_fails)          # {'TC1','TC3','TC5','TC7'}
print(run1_fails.union(run2_fails))     # same

# Intersection — consistently failing
print(run1_fails & run2_fails)           # {'TC3', 'TC5'}

# Difference — only in run1, not run2
print(run1_fails - run2_fails)           # {'TC1'}

# Symmetric difference — not in both
print(run1_fails ^ run2_fails)           # {'TC1', 'TC7'}

# Subset / superset check
print({"TC3"}.issubset(run1_fails))      # True
Deleting from sets + edge cases
s = {"TC1", "TC2", "TC3"}

s.remove("TC2")    # ❌ raises KeyError if missing!
s.discard("TC99") # ✅ safe — no error if missing
popped = s.pop()   # removes and returns ARBITRARY item
s.clear()          # removes all → empty set

# ❌ Sets can only contain HASHABLE types
# bad = {[1,2,3]}  ← TypeError: unhashable type: 'list'
# bad = {{"a":1}}  ← TypeError: unhashable type: 'dict'

# ✅ Tuples are hashable — can be set members
good = {(1,2), (3,4)}   # valid set of tuples
Interview one-liners
  • 1"A set is an unordered collection of unique hashable items — the go-to structure for deduplication and fast membership testing."
  • 2"Empty set must be set(), not {} — empty curly braces create an empty dict, a very common Python beginner mistake."
  • 3"Set intersection (A & B) finds tests that failed in ALL runs; set difference (A - B) finds tests that regressed only in one run."
  • 4"discard() is safer than remove() — it silently ignores missing elements, while remove() raises KeyError."
  • 5"Sets do not support indexing because items are stored by hash position, not sequential order — use list(my_set) to convert when you need ordering."
Interview prep Q&A
Q1: What is the time complexity of set membership? +
O(1) average case. Sets use a hash table internally — looking up an element requires computing its hash and checking one bucket, regardless of how many elements are in the set. List membership (in) is O(n). For QA: if you check hundreds of test IDs against a known list, convert to a set first.
Q2: Why can't you add a list to a set? +
Sets require hashable elements. Hashability requires an object to have an immutable identity — lists are mutable, so their hash would change if modified, breaking the set's internal structure. Solution: convert the list to a tuple first.
Q3: How would you find tests that passed in run 1 but failed in run 2? +
Use set difference: regressions = run2_fails - run1_fails. This returns tests in run2_fails that are NOT in run1_fails — the newly broken tests. Similarly, run1_fails - run2_fails gives tests that were fixed.
Q4: What is frozenset? +
An immutable version of set. Like tuples for lists, frozensets are hashable — they can be dict keys or members of another set. frozenset({1,2,3}). Useful when you need set math operations but the set should not be modified after creation.
Q5: When would you choose a set over a list? +
When: (1) you need uniqueness guaranteed, (2) membership testing is frequent (O(1) vs O(n)), (3) you need set math (union/intersection/difference). Keep a list when order matters, duplicates are valid, or you need index-based access.

S28 — Dictionary Collection: Complete Operations Guide
Key-value pairs · comprehension · nested dicts · all CRUD operations · JSON parallel · edge cases
Phase 2 — Collections
What is a dict?
An ordered (Python 3.7+), mutable mapping of unique keys to values. The most important Python collection for QA — every JSON API response, config file, and test data record maps to a dict.
Keys must be hashable
Strings, numbers, and tuples make valid keys. Lists and dicts cannot be keys. Values can be anything — nested dicts, lists, objects, None.
Creating dictionaries
# Empty dict
empty  = {}
empty2 = dict()

# Literal syntax
user = {"id": 1, "name": "Alice", "role": "admin"}

# dict() constructor
config = dict(host="localhost", port=5432, db="qa_db")

# Dict comprehension
status_map = {code: "ok" if code < 400 else "err"
              for code in (200, 201, 404, 500)}
# {200: 'ok', 201: 'ok', 404: 'err', 500: 'err'}

# Nested dict
test = {"id": "TC001", "env": {"url": "https://qa.com", "port": 443}}
Accessing items
user = {"id": 101, "name": "Alice", "role": "admin", "active": True}

# Direct access — KeyError if key missing
print(user["name"])                     # Alice

# .get() — safe, returns default if missing
print(user.get("email", "N/A"))         # N/A

# Views
print(list(user.keys()))               # ['id','name','role','active']
print(list(user.values()))             # [101,'Alice','admin',True]
print(list(user.items()))              # [(key,val),...]

# Loop key-value
for k, v in user.items():
    print(f"  {k}: {v}")

# Nested access
print(test["env"]["url"])              # chained access
Modifying dictionaries
config = {"timeout": 30, "retries": 3}

# Add / update key
config["env"]     = "staging"   # add new key
config["timeout"] = 60          # update existing

# .update() — merge another dict
overrides = {"timeout": 90, "browser": "chrome"}
config.update(overrides)

# Python 3.9+: merge operator |
defaults = {"timeout": 30, "retries": 3}
merged   = defaults | overrides   # new dict, overrides wins
print(merged)
# {'timeout': 90, 'retries': 3, 'browser': 'chrome'}

# setdefault — add key only if not present
config.setdefault("log_level", "INFO")
Deleting from dicts
user = {"id": 1, "name": "Alice", "temp": "remove_me"}

del user["temp"]                   # delete by key (KeyError if missing)

name = user.pop("name")           # remove + return value
print(name)                        # Alice

safe = user.pop("missing", "N/A") # safe pop with default

last = user.popitem()             # removes + returns last inserted (key,val)
print(last)                        # ('id', 1)

user.clear()                       # remove all → {}
Edge cases
# ❌ KeyError — accessing missing key directly
d = {"a": 1}
# d["b"]  ← KeyError: 'b'
# ✅ Always use .get() for API response parsing
val = d.get("b", 0)   # safe — returns 0

# ❌ Modifying dict size during iteration
d = {"a":1, "b":2, "c":3}
# for k in d: del d[k]  ← RuntimeError!
# ✅ Iterate over a copy of keys
for k in list(d.keys()):
    if d[k] < 2: del d[k]

# ❌ Unhashable key
# d[[1,2]] = "val"  ← TypeError: unhashable type: 'list'
# ✅ Use tuple as key instead
d[(1,2)] = "valid key"

  • 1"dict.get(key, default) is always safer than dict[key] when parsing JSON API responses — it never raises KeyError."
  • 2"Dictionaries are ordered in Python 3.7+ — items are iterated in insertion order, which makes them predictable for test configuration loading."
  • 3"The | merge operator (Python 3.9+) creates a new merged dict without mutating either operand — use it to merge default configs with environment overrides."
  • 4"pop(key, default) removes and returns a value safely — use it over del when you need the removed value or want to avoid KeyError."
  • 5"Dict comprehension {k: v for k, v in items if condition} builds a filtered or transformed dict in one readable line."
Interview prep Q&A
Q1: How do you safely access deeply nested dict keys? +
Chain .get(): data.get("user", {}).get("address", {}).get("city", "Unknown"). If any level is missing, get() returns {} which also responds to .get() safely. Or use a try/except for complex structures. Third-party: glom library for deep access.
Q2: What is a defaultdict and when is it useful? +
from collections import defaultdict; d = defaultdict(list) — accessing a missing key automatically creates it with the factory's default value instead of raising KeyError. Useful in QA for grouping test results: results["PASS"].append(tc_id) without checking if "PASS" key exists.
Q3: What is the difference between dict.update() and the | operator? +
a.update(b) mutates a in place (modifies original). a | b returns a new dict without touching a or b. Use |= for in-place merge: a |= b. In QA, prefer | when building config — preserve originals for test isolation.
Q4: How do you sort a dictionary by value? +
sorted(d.items(), key=lambda x: x[1]) returns a list of (key, value) tuples sorted by value. To get a sorted dict: dict(sorted(d.items(), key=lambda x: x[1])). Used in QA to rank test durations or failure counts.
Q5: How do you convert a JSON API response to a Python dict? +
import json; data = json.loads(response.text) or data = response.json() (requests library). The JSON object becomes a Python dict, arrays become lists, null becomes None, true/false become True/False. Always validate keys with .get() before accessing.
