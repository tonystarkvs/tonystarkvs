Session 3 · Core Definition

An iframe (inline frame) is an HTML element that embeds another HTML document inside the current page. Playwright provides frameLocator() to locate elements inside iframes using a CSS/XPath selector for the frame itself, then chaining locators for elements inside it.

WHAT is it?

An iframe creates a nested browsing context — a separate DOM inside the parent page. Playwright's frameLocator() returns a FrameLocator that scopes all subsequent locators to inside that iframe's document.

WHY handle it?

Many apps embed third-party widgets in iframes: payment forms (Stripe, PayPal), CAPTCHA, video players, map embeds, rich text editors (TinyMCE, CKEditor), chat widgets. Automating them requires frame handling.

WHEN to use it?

Any time the element you need to interact with is inside an iframe. You can tell by inspecting the DOM — if an element is inside an <iframe> tag, you need frameLocator() to reach it.

HOW to use it?

Use page.frameLocator('iframe selector') to get a FrameLocator, then chain .locator(), .getByRole(), .getByText() etc. on it exactly like a regular page. All actions (fill, click, check) work the same way inside frames.

WHERE does it fit in architecture?

Page → frameLocator('#payment-iframe') → locator('input[name="card"]'). Playwright handles the cross-frame boundary automatically. For nested iframes: page.frameLocator('outer') → .frameLocator('inner') → .locator('element').

Keyword Glossary — Session 3
frameLocator()Returns a FrameLocator object scoped to the iframe matching the selector. All locators chained on it resolve inside the frame's document. This is the modern Playwright API for frames.
FrameLocatorA lazily-evaluated reference to an iframe. Supports all locator methods: .locator(), .getByRole(), .getByText(), .getByLabel(), .first(), .nth(). Does not make a network request itself.
page.frames()Returns an array of all Frame objects on the page (including the main frame). Useful for finding a frame by name or URL using the legacy Frame API.
page.frame()Returns a Frame object by name attribute or URL. e.g. page.frame({name: 'payment'}) or page.frame({url: /stripe/}). Legacy API — prefer frameLocator() for new code.
FrameThe underlying Frame object representing an iframe or the main page frame. Has its own navigate(), fill(), click(), evaluate() etc. methods. Accessible via page.mainFrame() or page.frames().
mainFrame()Returns the Frame object for the main page document. All actions on `page` delegate to page.mainFrame() internally.
frame.childFrames()Returns an array of child frames (nested iframes) inside the current frame. Used to traverse a tree of nested iframes programmatically.
frame.url()Returns the URL currently loaded in this frame. Useful for identifying cross-origin frames by their domain.
frame.name()Returns the name attribute of the iframe element. e.g. <iframe name="payment"> → frame.name() returns 'payment'.
cross-origin frameAn iframe whose src is a different domain than the parent page. Playwright can interact with cross-origin iframes using frameLocator() — the same API works regardless of origin policy.
frameLocator.nth()When multiple iframes match the selector, use .nth(0), .nth(1) to target a specific one. Zero-indexed.
Interview One-Liners — Session 3
Modern API"In modern Playwright, frameLocator() is the preferred way to handle iframes — you chain locators on it exactly like a regular page, so there's no context switching, no frame reference management, and auto-waiting works exactly the same."
vs Selenium"In Selenium you call driver.switchTo().frame() to change the global driver context to inside the iframe, do your action, then must call driver.switchTo().defaultContent() to get back — Playwright has no such global state; frameLocator() is always scoped."
Nested Frames"For nested iframes, I chain frameLocator() calls: page.frameLocator('outer-iframe').frameLocator('inner-iframe').locator('button') — each frameLocator() drills one level deeper into the frame tree."
Cross-Origin"Playwright handles cross-origin iframes natively — frameLocator() works the same way regardless of whether the iframe is same-origin or cross-origin, unlike browser security policies that block JavaScript access."
Payment Forms"The most common real-world frame handling is payment forms — Stripe's card input is inside a cross-origin iframe. I use page.frameLocator('iframe[name=\"__privateStripeFrame\"]').locator('[placeholder=\"Card number\"]') to fill it."
Detection"I identify iframe elements in DevTools by looking for <iframe> tags in the DOM — once I find the frame's selector (by id, name, src, or title attribute), I can immediately use frameLocator() to scope my locators to it."
Rich Text Editors"TinyMCE and CKEditor render their editable area inside an iframe with designMode enabled. I use frameLocator('iframe.tox-edit-area__iframe').locator('body') and then .fill() or .type() to enter content into these rich text editors."
Core Concepts — Session 3

⬡ Frame Tree Architecture

Every page has a main frame. Iframes create child frames. Each child can have its own children — forming a tree. Playwright's frameLocator() lets you navigate this tree with chained calls.

Page (main frame)url: myapp.com
iframe#paymenturl: stripe.com/v3/elements
input[placeholder="Card number"]
iframe#recaptchaurl: google.com/recaptcha
iframe#chat-widgeturl: intercom.io
iframe#chat-innernested iframe!

⬡ frameLocator() vs Legacy frame() API

Playwright has two frame APIs. Always prefer frameLocator() for new code — it supports auto-waiting and works with all modern locator methods.

❌ LEGACY: page.frame({name:'payment'})
Returns Frame object synchronously — no auto-waiting
Must use await frame.fill(), frame.click() directly
Frame might not exist yet → throws immediately
✅ MODERN: page.frameLocator('iframe[name="payment"]')
Returns FrameLocator — lazy, supports auto-waiting
Chain .locator(), .getByRole() etc. — same as page
Waits for iframe to appear in DOM automatically

⬡ How to Identify Iframes in DevTools

Open DevTools → Elements tab. Look for <iframe> tags. Common selectors to identify them:

by id→iframe#payment-frame
by name attr→iframe[name="stripe"]
by src→iframe[src*="stripe.com"]
by title→iframe[title="Payment Form"]
by class→iframe.payment-widget
nth of type→page.frameLocator('iframe').nth(0)
Best Practices — Session 3
  • ✓
    Always use frameLocator() over the legacy frame() API — it supports auto-waiting, works with all modern locator methods, and eliminates frame-not-found errors on slow-loading iframes.
  • ✓
    Identify frames by stable attributes like id, name, or title — never by array index from page.frames() alone, as iframe order can change when the page renders differently.
  • ✓
    For rich text editors (TinyMCE, CKEditor): use frameLocator to get the iframe, then locator('body') to target the editable content area — the body element itself receives the text input.
  • ✓
    For Stripe payment iframes: each card field (number, expiry, CVC) is a separate iframe. Use frameLocator with the specific field's iframe selector for each one — don't assume one iframe contains all fields.
  • ✓
    Use frame.waitForLoadState() on the Frame object if you need to ensure the iframe's content has fully loaded before proceeding with assertions inside it.
  • ✓
    For CAPTCHA iframes: in test environments, configure your app to disable CAPTCHA rather than trying to automate it. If you must test the UI, verify the iframe is present without solving it.
  • ✓
    Use page.frames() to debug — log all frame URLs and names to understand the frame tree before writing selectors.
Interview Prep — 5 SDET Q&A
Q1: What is the difference between frameLocator() and page.frame() in Playwright? ▶
page.frame({name, url}) is the legacy API — it returns a Frame object synchronously without waiting. If the iframe hasn't loaded yet, it returns null immediately. You then call methods directly on the Frame (frame.fill(), frame.click()). frameLocator() is the modern API — it returns a FrameLocator which is lazy and supports auto-waiting. It integrates with Playwright's locator system: you can chain getByRole(), getByLabel(), filter(), nth() etc. on it. The FrameLocator waits for the iframe to appear in the DOM before acting. For new tests, always use frameLocator().
Q2: How do you handle Stripe's payment iframe in Playwright? ▶
Stripe renders each field (card number, expiry, CVC) in a separate iframe. I inspect the DOM to find each iframe's selector. Then: const cardFrame = page.frameLocator('iframe[name="__privateStripeFrame__cardNumber"]'). await cardFrame.locator('[placeholder="Card number"]').fill('4242424242424242'). For expiry: const expiryFrame = page.frameLocator('iframe[name="__privateStripeFrame__cardExpiry"]'). await expiryFrame.locator('[placeholder="MM / YY"]').fill('12/28'). The exact iframe name attributes depend on the Stripe Elements version — always inspect the live DOM. Playwright handles cross-origin frames transparently.
Q3: How do you interact with a TinyMCE rich text editor inside an iframe? ▶
TinyMCE renders an iframe with a body element that has contenteditable or designMode enabled. First find the iframe: const editorFrame = page.frameLocator('iframe.tox-edit-area__iframe') — the class name varies by TinyMCE version, check DevTools. Then target the body: await editorFrame.locator('body').fill('My test content'). If fill() doesn't work (some editors intercept it), use .click() first then page.keyboard.type('My test content'). For formatted input (bold, lists), I click the toolbar buttons in the main page, not inside the iframe — toolbar buttons are in the parent DOM.
Q4: How do you handle nested iframes in Playwright? ▶
I chain frameLocator() calls. If there's an outer iframe containing an inner iframe: const outerFrame = page.frameLocator('iframe#outer'). const innerFrame = outerFrame.frameLocator('iframe#inner'). await innerFrame.locator('button#submit').click(). Each frameLocator() call descends one level. Alternatively with the legacy API: const outerFrameObj = page.frame({name: 'outer'}). const innerFrameObj = outerFrameObj.childFrames()[0]. I prefer the chained frameLocator() approach as it's cleaner and supports auto-waiting at each level.
Q5: How do you debug an unknown frame structure you've never seen before? ▶
My debugging workflow: First, console.log all frames: console.log(page.frames().map(f => ({name: f.name(), url: f.url()})). This reveals every iframe's name and URL. Second, I use DevTools to inspect the element I want — if it's inside an iframe, I look at the parent iframe's id, name, src, and title attributes. Third, I try frameLocator with different selectors until one resolves. Fourth, I log the frame tree with frame.childFrames() recursively to understand nesting. Finally I use page.pause() in headed mode and the Playwright Inspector to visually inspect and generate locators interactively.

basic-frame.ts
// basic-frame.ts
// PURPOSE: Interact with elements inside an iframe using frameLocator()
// USE CASE: Any embedded widget, embedded form, or embedded content

import { test, expect } from '@playwright/test';

test('interact with elements inside iframe', async ({ page }) => {

  await page.goto('https://myapp.com/checkout');

  // ── Locate the iframe first ──────────────────────────────────────
  // frameLocator() accepts any CSS selector that points to the <iframe> element
  const paymentFrame = page.frameLocator('iframe#payment-widget');

  // ── All locator methods work inside the frame ────────────────────
  // Fill card number input inside the iframe
  await paymentFrame.locator('input[name="cardNumber"]').fill('4111111111111111');

  // Use getByLabel (Playwright auto-waits for the frame to load)
  await paymentFrame.getByLabel('Expiry Date').fill('12/28');
  await paymentFrame.getByLabel('CVV').fill('123');

  // Use getByRole inside frame
  await paymentFrame.getByRole('button', { name: 'Pay Now' }).click();

  // Assert success message inside the frame
  await expect(
    paymentFrame.locator('.success-message')
  ).toContainText('Payment successful');

  // You can also assert on the parent page simultaneously
  await expect(page.locator('h1')).toContainText('Checkout');
});
stripe-payment.ts
// stripe-payment.ts
// PURPOSE: Fill Stripe Elements payment form (multiple iframes)
// INSIGHT: Each Stripe field is a SEPARATE iframe

import { test, expect } from '@playwright/test';

test('complete Stripe payment with card details', async ({ page }) => {

  await page.goto('https://myshop.com/checkout');

  // Debug: Log all iframes to find their selectors
  const allFrames = page.frames();
  allFrames.forEach(f => console.log(f.name(), f.url()));

  // ── Stripe card number field (separate iframe) ───────────────────
  const cardNumberFrame = page.frameLocator(
    'iframe[name*="__privateStripeFrame"][name*="cardNumber"]'
  );
  await cardNumberFrame
    .locator('[placeholder="Card number"]')
    .fill('4242 4242 4242 4242'); // Stripe test card

  // ── Stripe expiry field (a different iframe!) ────────────────────
  const expiryFrame = page.frameLocator(
    'iframe[name*="__privateStripeFrame"][name*="cardExpiry"]'
  );
  await expiryFrame
    .locator('[placeholder="MM / YY"]')
    .fill('12 / 28');

  // ── Stripe CVC field (yet another iframe!) ───────────────────────
  const cvcFrame = page.frameLocator(
    'iframe[name*="__privateStripeFrame"][name*="cardCvc"]'
  );
  await cvcFrame
    .locator('[placeholder="CVC"]')
    .fill('424');

  // Submit button is on PARENT page, not inside iframe
  await page.click('button#pay-button');

  // Assert success on parent page
  await page.waitForURL('**/order-success');
  await expect(page.locator('.order-confirmed')).toBeVisible();
});

// nested-frames.ts
// PURPOSE: Navigate nested iframes (iframe inside an iframe)
// USE CASE: Complex embeds like analytics dashboards or ad tech

import { test, expect } from '@playwright/test';

test('interact with deeply nested iframe', async ({ page }) => {

  await page.goto('https://myapp.com/dashboard');

  // Level 1: Outer iframe (e.g. analytics widget container)
  const outerFrame = page.frameLocator('iframe#analytics-widget');

  // Level 2: Inner iframe (e.g. chart embedded inside analytics widget)
  const innerFrame = outerFrame.frameLocator('iframe.chart-container');

  // Interact with element deep inside nested iframes
  await innerFrame.locator('.chart-legend-item').first().click();
  await expect(innerFrame.locator('.tooltip')).toBeVisible();

  // ── Using Legacy API for nested frames ────────────────────────────
  const mainFrame = page.mainFrame();
  const outerFrameObj = mainFrame.childFrames()
    .find(f => f.name() === 'analytics-widget');

  if (outerFrameObj) {
    const childFrames = outerFrameObj.childFrames();
    console.log(`Child frames inside analytics: ${childFrames.length}`);
    childFrames.forEach(cf =>
      console.log(`  ↳ ${cf.name()} — ${cf.url()}`)
    );
  }
});

debug-frames.ts
// debug-frames.ts
// PURPOSE: Debug utilities to discover all frames on a page
// USE CASE: When you first encounter an unfamiliar page with unknown iframes

import { test, expect, Frame } from '@playwright/test';

// Recursive function to print the entire frame tree
function printFrameTree(frame: Frame, indent = 0): void {
  const prefix = '  '.repeat(indent);
  console.log(`${prefix}📄 name="${frame.name()}" url=${frame.url()}`);
  frame.childFrames().forEach(child => printFrameTree(child, indent + 1));
}

test('discover all iframes on a page', async ({ page }) => {

  await page.goto('https://myapp.com/checkout');
  await page.waitForLoadState('networkidle');

  // Method 1: Print flat list of all frames
  console.log('\n── All Frames (flat) ──');
  page.frames().forEach((f, i) => {
    console.log(`[${i}] name="${f.name()}" url="${f.url()}"`);
  });

  // Method 2: Print full tree with nesting
  console.log('\n── Frame Tree ──');
  printFrameTree(page.mainFrame());

  // Method 3: Find frame by URL substring
  const stripeFrame = page.frames().find(f => f.url().includes('stripe.com'));
  if (stripeFrame) console.log('Found Stripe frame!', stripeFrame.name());

  // Method 4: Count total iframes
  const iframeCount = page.frames().length - 1; // -1 for main frame
  console.log(`Total iframes: ${iframeCount}`);
});
legacy-frame-api.ts
// legacy-frame-api.ts
// PURPOSE: Legacy Frame API usage (page.frame() / page.mainFrame())
// NOTE: Prefer frameLocator() for new code — this shows legacy for reference

import { test, expect, Frame } from '@playwright/test';

test('using legacy Frame API', async ({ page }) => {

  await page.goto('https://myapp.com/embed');

  // Get frame by name attribute: <iframe name="myFrame">
  const frameByName: Frame | null = page.frame({ name: 'myFrame' });

  // Get frame by URL pattern
  const frameByURL: Frame | null = page.frame({ url: /embed\.myservice\.com/ });

  // Always null-check — frame() returns null if not found
  if (!frameByName) throw new Error('Frame "myFrame" not found!');

  // Use Frame methods directly (note: these don't auto-wait like frameLocator)
  await frameByName.fill('input#search', 'playwright testing');
  await frameByName.click('button[type="submit"]');

  // Frame.locator() → returns a standard Locator (supports auto-waiting)
  await expect(
    frameByName.locator('.results-count')
  ).toContainText('results');

  // Frame evaluate: run JS inside the frame's context
  const frameTitle = await frameByName.evaluate(() => document.title);
  console.log(`Frame document title: ${frameTitle}`);

  // Access frame's URL and name
  console.log(`Frame URL: ${frameByName.url()}`);
  console.log(`Frame name: ${frameByName.name()}`);

  // Parent frame access
  const parent = frameByName.parentFrame();
  console.log(`Parent frame: ${parent?.url()}`);
});
