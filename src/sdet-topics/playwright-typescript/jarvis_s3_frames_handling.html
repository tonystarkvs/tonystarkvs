Overview — Frames & Iframes Handling in Playwright

An iframe (inline frame) is an HTML element that embeds another HTML document inside the current page. Playwright provides frameLocator() to locate elements inside iframes using a CSS/XPath selector for the frame itself, then chaining locators for elements inside it. Unlike Selenium where you must switchTo().frame() and then defaultContent(), Playwright's frameLocator() is always scoped — no global state switching needed.

What / Why / When / Where / How

WhatAn iframe creates a nested browsing context — a separate DOM inside the parent page. Playwright's frameLocator() returns a FrameLocator that scopes all subsequent locators to inside that iframe's document.
WhyMany apps embed third-party widgets in iframes: payment forms (Stripe, PayPal), CAPTCHA, video players, map embeds, rich text editors (TinyMCE, CKEditor), chat widgets. Automating them requires frame handling.
WhenAny time the element you need to interact with is inside an iframe. You can tell by inspecting the DOM — if an element is inside an <iframe> tag, you need frameLocator() to reach it.
WherePage → frameLocator('#payment-iframe') → locator('input[name="card"]'). Playwright handles the cross-frame boundary automatically. For nested iframes: page.frameLocator('outer') → .frameLocator('inner') → .locator('element').
HowUse page.frameLocator('iframe selector') to get a FrameLocator, then chain .locator(), .getByRole(), .getByText() etc. on it exactly like a regular page. All actions (fill, click, check) work the same way inside frames.

Keyword Glossary

frameLocator()
Returns a FrameLocator object scoped to the iframe matching the selector. All locators chained on it resolve inside the frame's document. This is the modern Playwright API for frames.
FrameLocator
A lazily-evaluated reference to an iframe. Supports all locator methods: .locator(), .getByRole(), .getByText(), .getByLabel(), .first(), .nth(). Does not make a network request itself.
page.frames()
Returns an array of all Frame objects on the page (including the main frame). Useful for finding a frame by name or URL using the legacy Frame API.
page.frame()
Returns a Frame object by name attribute or URL. e.g. page.frame({name: 'payment'}) or page.frame({url: /stripe/}). Legacy API — prefer frameLocator() for new code.
Frame
The underlying Frame object representing an iframe or the main page frame. Has its own navigate(), fill(), click(), evaluate() methods. Accessible via page.mainFrame() or page.frames().
mainFrame()
Returns the Frame object for the main page document. All actions on page delegate to page.mainFrame() internally.
frame.childFrames()
Returns an array of child frames (nested iframes) inside the current frame. Used to traverse a tree of nested iframes programmatically.
frame.url()
Returns the URL currently loaded in this frame. Useful for identifying cross-origin frames by their domain.
frame.name()
Returns the name attribute of the iframe element. e.g. <iframe name="payment"> → frame.name() returns 'payment'.
cross-origin frame
An iframe whose src is a different domain than the parent page. Playwright can interact with cross-origin iframes using frameLocator() — the same API works regardless of origin policy.
frameLocator.nth()
When multiple iframes match the selector, use .nth(0), .nth(1) to target a specific one. Zero-indexed.

Interview One-Liners

In modern Playwright, frameLocator() is the preferred way to handle iframes — you chain locators on it exactly like a regular page, so there's no context switching or frame reference management.
In Selenium you call driver.switchTo().frame() and then defaultContent() — Playwright has no such global state; frameLocator() is always scoped.
For nested iframes, I chain frameLocator() calls: page.frameLocator('outer-iframe').frameLocator('inner-iframe').locator('button') to drill down.
Playwright handles cross-origin iframes natively — frameLocator() works the same way regardless of same-origin or cross-origin restrictions.
Stripe's card input is inside a cross-origin iframe. I use page.frameLocator('iframe[name*="__privateStripeFrame"]') to fill it.
I identify iframe elements in DevTools by looking for <iframe> tags and finding a stable id, name, src, or title selector.
TinyMCE and CKEditor render their editable area inside an iframe. I use frameLocator('iframe.tox-edit-area__iframe').locator('body').fill('text') to input content.

Frame Tree Architecture
Main FrameEvery page has a main frame. Iframes create child frames. Each child can have its own children — forming a tree.
frameLocator()Modern API — lazy, supports auto-waiting. Chain .locator(), .getByRole() etc. Waits for iframe to appear in DOM.
Legacy frame()Returns Frame object synchronously — no auto-waiting. Frame might not exist yet → returns null or throws immediately.
Cross-originPlaywright handles cross-origin iframes transparently — same frameLocator() API works regardless of origin policy.
