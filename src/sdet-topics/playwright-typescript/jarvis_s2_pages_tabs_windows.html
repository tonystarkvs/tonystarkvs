Session 2 · Core Definition

A Page in Playwright represents a single browser tab or window. Pages live inside a BrowserContext. You can open new tabs programmatically, detect popup windows triggered by user actions (like clicking links with target="_blank"), and control focus between multiple open pages.

WHAT is it?

A Page object wraps one browser tab. It exposes all navigation, DOM interaction, and event APIs. Multiple pages can exist inside one context, sharing cookies/auth but each with its own URL and DOM state.

WHY use it?

Modern apps open new tabs for OAuth flows, receipts, PDFs, and external links. Playwright must capture and control those popups — missing them means your test goes blind to key user journeys.

WHEN to use it?

Whenever your app opens a new tab (target="_blank" links, window.open() calls, OAuth redirects, download confirmations) or when you need to test multi-tab workflows like comparing two products.

HOW to detect popups?

Use page.waitForEvent('popup') BEFORE triggering the action that opens the new tab. This sets up the listener first, then the click happens, and Playwright captures the new page reference.

WHERE does it fit?

Browser → BrowserContext → Page (Tab 1) + Page (Tab 2 — popup). Both tabs share the same context cookies. Switching between them is done by simply using the correct page variable — no "switch to window handle" like Selenium.

Keyword Glossary — Session 2
context.newPage()Creates a new tab programmatically inside the current BrowserContext. Returns a Page object. The new page shares cookies/auth with the context.
page.waitForEvent('popup')Listens for the 'popup' event which fires when a new tab/window opens from this page (via target="_blank" or window.open). Must be called BEFORE the action that triggers the popup.
popupA new Page instance that opened as a result of a user action on the parent page. Has its own URL, DOM, and navigation state. Returned by waitForEvent('popup').
page.bringToFront()Brings the specified page/tab to the foreground (focused). Useful when multiple tabs are open and you need to ensure UI interactions happen on the correct tab.
context.pages()Returns an array of all currently open Page objects inside the context. Useful for finding a specific tab by URL or title when you don't have a direct reference.
page.waitForLoadState()Waits for the page to reach a specific load state: 'load' (all resources), 'domcontentloaded' (HTML parsed), or 'networkidle' (no network requests for 500ms).
page.close()Closes a specific page/tab. Does NOT close the context or browser. Use to clean up popup pages after you're done asserting them.
page.url()Returns the current URL string of the page. Useful for asserting the popup landed on the correct URL after opening.
page.title()Returns the page's document title (the <title> tag content). Async method — must be awaited.
page.on('popup')Event listener variant of popup detection. Alternative to waitForEvent — registers a callback that fires every time a new tab opens from this page.
window.open()JavaScript method in the browser that opens a new tab/window. Playwright detects this as a 'popup' event. Can be triggered by JS code or button clicks.

Core Concepts — Session 2

⬡ How Popup Detection Works (Event Race Condition)

The most common beginner mistake with popups is the ordering problem. Playwright's 'popup' event fires the moment the new tab opens. If you await the click first, the event has already fired before your listener is registered — you miss it forever.

❌ WRONG ORDER
await page.click('#open-tab')→popup fires NOW
await page.waitForEvent('popup')→too late! missed it
✅ CORRECT ORDER
const popupPromise = page.waitForEvent('popup')→listener ready
await page.click('#open-tab')→popup fires
const popup = await popupPromise→captured! ✓

⬡ Page Lifecycle States

After navigating or getting a popup reference, the page goes through several states. Waiting for the wrong state causes flaky tests.

⬡ Multi-Tab Architecture in Playwright

All pages in a context are independent tab objects. You control them by variable reference — not by "switching active tab" like Selenium. Both pages run concurrently.

BrowserContext
page1 → Tab 1 (parent)active, navigatable
page2 → Tab 2 (popup)active, navigatable
page3 → Tab 3 (manual)via context.newPage()
page.bringToFront()→focuses that tab in UI
context.pages()→[page1, page2, page3]
Best Practices — Session 2
  • ✓
    Always register waitForEvent('popup') BEFORE the click that triggers it. Store it as a Promise, click, then await the Promise — this is the correct race-condition-safe pattern.
  • ✓
    After capturing a popup reference, always await popup.waitForLoadState() before interacting. The reference is returned before the page DOM is ready.
  • ✓
    Use context.pages() to find a tab dynamically by URL or title when you lose the reference — never hardcode an array index like context.pages()[1] in production tests.
  • ✓
    Always close popup pages after asserting them with await popup.close() — leaving them open bloats context.pages() and can cause unexpected interactions with event listeners.
  • ✓
    Use Promise.all() when opening multiple tabs simultaneously — this lets them load in parallel instead of sequentially, making tests faster.
  • ✓
    For OAuth flows that open a popup for login: capture the popup, fill credentials in the popup page, then assert the parent page reflects the logged-in state — the context shares the resulting auth cookies automatically.
  • ✓
    Call page.bringToFront() before clipboard or focus-sensitive interactions — some browsers silently fail these operations on background tabs.
Interview Prep — 5 SDET Q&A
Q1: How does Playwright handle new tab/window popups differently from Selenium? ▶
In Selenium, you manage window handles — you call getWindowHandles() to get a Set of IDs, then switchTo().window(handle) to change focus. This is fragile because handle IDs are opaque strings and switching is a global state change. In Playwright, each new tab returns a concrete Page object via waitForEvent('popup'). You interact with any tab at any time using its variable — there's no global "active window" concept. You can drive page1 and popupPage simultaneously without switching. This is a fundamental design improvement that eliminates an entire class of flaky test patterns.
Q2: Why must waitForEvent('popup') be called before the click, not after? ▶
Because 'popup' is an event, not a polling check. When you call waitForEvent('popup'), Playwright registers an event listener on the page. If the click happens first, the popup opens, the event fires immediately, and there's no listener to catch it — it's gone. The Promise returned by waitForEvent('popup') will then hang forever waiting for a popup that already happened. The correct pattern is: const popupPromise = page.waitForEvent('popup') — this returns a Promise that resolves when the event fires. Then you click. Then you await popupPromise. This guarantees the listener is registered before the event can fire.
Q3: How would you find a specific tab by URL when you have multiple tabs open? ▶
I use context.pages() which returns an array of all open Page objects in the context. Then I filter by URL: const targetPage = context.pages().find(p => p.url().includes('/dashboard')). If the page might still be loading, I'd instead do: const pages = context.pages(); const target = pages.find(p => p.url().includes('target-path')) ?? await context.waitForEvent('page'); This handles the case where the page hasn't been added to the array yet. I always validate the result is not undefined before using it.
Q4: How would you test an OAuth flow that opens a popup for Google login? ▶
First, I register the popup listener before clicking Login with Google: const popupPromise = page.waitForEvent('popup'). Then click the button: await page.click('#google-login-btn'). Capture the popup: const googlePopup = await popupPromise. Wait for it to load: await googlePopup.waitForLoadState('domcontentloaded'). Fill in credentials on the Google login page: await googlePopup.fill('#email', testEmail), await googlePopup.click('#next'), await googlePopup.fill('#password', testPass), await googlePopup.click('#signIn'). The popup closes automatically after Google redirects back. Then I assert on the parent page that the user is logged in: await expect(page.locator('.user-avatar')).toBeVisible(). The auth cookie is shared via the context.
Q5: What is page.bringToFront() and when is it necessary? ▶
bringToFront() makes a specific tab the active/focused tab in the browser UI. In headless mode this matters less, but in headed mode and for certain browser APIs it's critical. Clipboard operations (navigator.clipboard.writeText/readText) are often restricted to the focused tab — calling them on a background tab silently fails or throws a permissions error. Similarly, some drag-and-drop implementations and focus() calls behave differently on background tabs. I call page.bringToFront() before any clipboard test, before testing keyboard shortcuts that depend on window focus, and before testing any feature that explicitly checks document.hasFocus() in its source code.

new-tab-manual.ts
// new-tab-manual.ts
// PURPOSE: Open a new tab programmatically and work with it
// USE CASE: Test multi-step flows across tabs you control

import { test, expect } from '@playwright/test';

test('open new tab manually and navigate', async ({ context }) => {

  // Tab 1: Open first page
  const page1 = await context.newPage();
  await page1.goto('https://playwright.dev');
  await expect(page1).toHaveTitle(/Playwright/);

  // Tab 2: Open second tab in the same context (shares cookies)
  const page2 = await context.newPage();
  await page2.goto('https://example.com');

  // Both tabs are open simultaneously — verify independently
  await expect(page1).toHaveURL(/playwright\.dev/);
  await expect(page2).toHaveURL(/example\.com/);

  // Bring Tab 1 to front (focus it)
  await page1.bringToFront();

  // How many tabs are open in this context?
  const allPages = context.pages();
  console.log(`Open tabs: ${allPages.length}`); // → 2

  // Close Tab 2 explicitly
  await page2.close();
  console.log(`After close: ${context.pages().length} tabs`); // → 1
});
popup-detection.ts
// popup-detection.ts
// PURPOSE: Capture a popup opened by target="_blank" link or window.open()
// CRITICAL: listener MUST be set up BEFORE the triggering action

import { test, expect } from '@playwright/test';

test('detect and interact with popup tab', async ({ page }) => {

  await page.goto('https://myapp.com/products');

  // ✅ STEP 1: Register the popup listener FIRST (returns a Promise)
  const popupPromise = page.waitForEvent('popup');

  // ✅ STEP 2: Trigger the action that opens the new tab
  // This link has target="_blank" — opens in new tab
  await page.click('a[target="_blank"]');

  // ✅ STEP 3: Await the popup Promise — returns the new Page
  const popup = await popupPromise;

  // STEP 4: Wait for popup DOM to be ready before interacting
  await popup.waitForLoadState('domcontentloaded');

  // STEP 5: Assert on the popup page
  await expect(popup).toHaveURL(/product-detail/);
  await expect(popup.locator('h1.product-title')).toBeVisible();

  // STEP 6: Interact with the popup if needed
  await popup.click('#add-to-cart');

  // STEP 7: Close the popup — clean up
  await popup.close();

  // Parent page is unaffected and still open
  await expect(page).toHaveURL(/products/);
});
multi-tab-control.ts
// multi-tab-control.ts
// PURPOSE: Control multiple tabs simultaneously
// USE CASE: Compare two pages, test tab-switching UX

import { test, expect } from '@playwright/test';

test('compare product details across two tabs', async ({ context }) => {

  // Open two tabs in parallel (both load simultaneously)
  const [page1, page2] = await Promise.all([
    context.newPage().then(p => (p.goto('https://myapp.com/product/1'), p)),
    context.newPage().then(p => (p.goto('https://myapp.com/product/2'), p)),
  ]);

  // Wait for both to finish loading
  await Promise.all([
    page1.waitForLoadState('networkidle'),
    page2.waitForLoadState('networkidle'),
  ]);

  // Read prices from both tabs
  const price1 = await page1.locator('.price').textContent();
  const price2 = await page2.locator('.price').textContent();
  console.log(`Product 1: ${price1} | Product 2: ${price2}`);

  // Bring page1 to front and interact
  await page1.bringToFront();
  await page1.click('#add-to-cart');

  // Switch to page2 and interact
  await page2.bringToFront();
  await page2.click('#add-to-cart');

  // Verify cart count updated on page1 (shared context = shared cookies)
  await page1.reload();
  await expect(page1.locator('.cart-count')).toHaveText('2');

  // Close both tabs
  await Promise.all([page1.close(), page2.close()]);
});
oauth-popup.ts
// oauth-popup.ts
// PURPOSE: Handle OAuth / social login popup flow
// USE CASE: Google/GitHub login opens a popup window for auth

import { test, expect } from '@playwright/test';

test('login with Google OAuth popup', async ({ page }) => {

  await page.goto('https://myapp.com/login');

  // Register popup listener BEFORE clicking — critical ordering!
  const popupPromise = page.waitForEvent('popup');

  // Click "Login with Google" button (opens OAuth popup window)
  await page.click('#google-login-btn');

  // Capture the OAuth popup page
  const googlePopup = await popupPromise;

  // Wait for Google's login form to load
  await googlePopup.waitForLoadState('domcontentloaded');

  // Verify we landed on Google's auth page
  await expect(googlePopup).toHaveURL(/accounts\.google\.com/);

  // Fill in test Google credentials
  await googlePopup.fill('input[type="email"]', 'testuser@gmail.com');
  await googlePopup.click('#identifierNext');
  await googlePopup.waitForSelector('input[type="password"]');
  await googlePopup.fill('input[type="password"]', 'TestPass123!');
  await googlePopup.click('#passwordNext');

  // Google redirects back → popup closes automatically
  // Wait for parent page to show logged-in state
  await page.waitForURL('**/dashboard');

  // Auth cookies are shared via context — parent page is now logged in
  await expect(page.locator('.user-avatar')).toBeVisible();
  await expect(page.locator('.welcome-message'))
    .toContainText('testuser');
});
find-tab-by-url.ts
// find-tab-by-url.ts
// PURPOSE: Find a specific tab by URL or title
// USE CASE: When you don't have a direct reference to the page you need

import { test, expect, Page } from '@playwright/test';

// Helper: find page by URL substring
function findPageByURL(pages: Page[], urlPart: string): Page | undefined {
  return pages.find(p => p.url().includes(urlPart));
}

test('find and use a tab by its URL', async ({ context }) => {

  // Open three tabs with different pages
  const home   = await context.newPage();
  const about  = await context.newPage();
  const contact = await context.newPage();

  await Promise.all([
    home.goto('https://myapp.com/'),
    about.goto('https://myapp.com/about'),
    contact.goto('https://myapp.com/contact'),
  ]);

  // Find the contact tab dynamically using context.pages()
  const allPages = context.pages();
  console.log(`Total open tabs: ${allPages.length}`); // 3

  // Find contact tab without having the direct variable
  const contactTab = findPageByURL(allPages, 'contact');
  if (!contactTab) throw new Error('Contact tab not found!');

  // Use the found tab
  await contactTab.bringToFront();
  await expect(contactTab.locator('h1')).toContainText('Contact');

  // Find page by title (async — must await title())
  const titles = await Promise.all(allPages.map(p => p.title()));
  const aboutIndex = titles.findIndex(t => t.includes('About'));
  const aboutTab = allPages[aboutIndex];
  await expect(aboutTab).toHaveURL(/about/);
});
window-events.ts
// window-events.ts
// PURPOSE: Listen to page-level events (close, crash, popup, download)
// USE CASE: Track unexpected tab activity, verify cleanup

import { test, expect } from '@playwright/test';

test('listen to page events', async ({ context, page }) => {

  // Track ALL popups opened from context (global listener)
  const openedPopups: string[] = [];
  context.on('page', (newPage) => {
    openedPopups.push(newPage.url());
    console.log(`🆕 New tab opened: ${newPage.url()}`);
  });

  // Listen for page close event
  page.on('close', () => {
    console.log('🔴 Page was closed');
  });

  // Listen for page crash (e.g. OOM, renderer process crash)
  page.on('crash', () => {
    console.error('💥 Page crashed!');
  });

  // Listen for console errors on the page
  page.on('console', (msg) => {
    if (msg.type() === 'error') {
      console.error(`❌ Browser console error: ${msg.text()}`);
    }
  });

  // Navigate and trigger some popups
  await page.goto('https://myapp.com');

  // Programmatically open a popup via JS evaluation
  const popupPromise = page.waitForEvent('popup');
  await page.evaluate(() => window.open('https://example.com', '_blank'));
  const newTab = await popupPromise;
  await newTab.waitForLoadState();

  // Verify our context listener tracked the new page
  expect(openedPopups.length).toBeGreaterThan(0);
  console.log(`Tracked popups: ${openedPopups.join(', ')}`);

  await newTab.close();
});
