Session 4 · Core Definition

Browser dialogs are native browser pop-ups triggered by JavaScript: alert(), confirm(), prompt(), and the page unload warning (beforeunload). They block page execution until dismissed. Playwright handles them via the page.on('dialog', handler) event listener.

alert()

Shows a message with one OK button. No return value. Used to notify users. Action: dialog.accept()

confirm()

Shows a message with OK + Cancel. Returns true/false. Used for "Are you sure?" prompts. Actions: accept() or dismiss()

prompt()

Shows a message + text input. Returns user's text or null. Used to collect input. Action: dialog.accept(text)

beforeunload

Fires when user tries to leave page. Browser shows "Leave site?" warning. Action: dismiss() to stay, accept() to leave

WHY handle dialogs?

If a dialog appears and no handler is registered, Playwright auto-dismisses it by default. If your test needs to verify dialog content, accept it, or input text into a prompt — you must register a handler.

HOW to handle?

Register page.on('dialog', handler) BEFORE triggering the action that opens the dialog. The handler receives a Dialog object with methods: accept(), dismiss(), message(), type(), defaultValue().

Keyword Glossary — Session 4
page.on('dialog')Registers an event listener that fires whenever a browser dialog (alert/confirm/prompt/beforeunload) appears. The handler receives a Dialog object. Must be registered before the action that triggers the dialog.
DialogThe Playwright object representing an open browser dialog. Provides: type(), message(), defaultValue(), accept(), dismiss(). Dialogs block page execution until handled.
dialog.accept()Clicks the OK/Accept button on the dialog. For prompt dialogs, pass a string: dialog.accept('my input text') to fill the text field before accepting.
dialog.dismiss()Clicks the Cancel/Dismiss button. For alert dialogs (which have no Cancel), dismiss() behaves like accept(). For confirm/prompt, returns false/null to the calling JS code.
dialog.message()Returns the text content of the dialog. e.g. for alert('Are you sure?'), dialog.message() returns 'Are you sure?'. Use this to assert the dialog shows the correct message.
dialog.type()Returns the type of dialog as a string: 'alert', 'confirm', 'prompt', or 'beforeunload'. Useful in a single handler that processes multiple dialog types differently.
dialog.defaultValue()Returns the default value pre-filled in a prompt dialog's input field. e.g. prompt('Enter name', 'John') → defaultValue() returns 'John'.
page.waitForEvent('dialog')Alternative to page.on() — returns a Promise that resolves with the Dialog object when it appears. Useful when you need to handle a dialog exactly once without a persistent listener.
beforeunloadA special dialog type that fires when the user navigates away from a page with unsaved changes. The app calls window.onbeforeunload. Browser shows a warning. dialog.accept() lets the user leave; dismiss() keeps them on the page.
page.removeListener()Removes a previously registered event listener. Use to remove dialog handlers after a specific test scenario so they don't interfere with subsequent test steps.
Interview One-Liners — Session 4
Default Behavior"Playwright auto-dismisses dialogs by default if no handler is registered — this means your test won't hang on an unexpected alert, but it also means you won't see the dialog message or verify its content unless you set up page.on('dialog')."
Ordering"Just like popup detection, dialog handling requires the listener to be set up BEFORE the action that triggers the dialog — otherwise you miss the event and the dialog auto-dismisses before your code can inspect it."
Types"There are four browser dialog types in Playwright: alert (one button), confirm (two buttons — OK/Cancel), prompt (text input), and beforeunload (leave/stay warning) — dialog.type() tells you which one you're dealing with."
Prompt"For prompt dialogs, dialog.accept('my text') fills the text field and clicks OK in one call — you don't interact with the input separately, you just pass the value directly to accept()."
beforeunload"Testing the beforeunload dialog is tricky — I register the handler, then call page.close() or navigate away, and the handler dismisses or accepts it. Just navigating with goto() doesn't always trigger it depending on app implementation."
Assertion"I always assert dialog.message() in my handler before calling accept() or dismiss() — this verifies the app is showing the right message text, not just that a dialog appeared."
Multiple Dialogs"If a test scenario involves multiple sequential dialogs, I use page.on() with a counter or a queue of responses — each call to the handler consumes one dialog and pops the next expected response."
Core Concepts — Session 4

⬡ Dialog Event Flow

Dialogs are synchronous in the browser — they block all JS execution. Playwright intercepts them via the 'dialog' event and lets you handle them asynchronously in your test code.

Test registers page.on('dialog', handler)
↓
Test action triggers dialog (click delete btn)
↓
Browser fires 'dialog' event
↓
Playwright calls handler(dialog)
↓
handler: assert message → accept() or dismiss()
↓
Browser dialog closes → page JS continues

⬡ Dialog Types — Accept vs Dismiss Behavior

alert('msg')→accept()→OK clicked, no return
alert('msg')→dismiss()→same as accept (no cancel btn)
confirm('sure?')→accept()→JS gets true
confirm('sure?')→dismiss()→JS gets false
prompt('name?', 'default')→accept('Alice')→JS gets 'Alice'
prompt('name?', 'default')→dismiss()→JS gets null
beforeunload→accept()→user leaves page
beforeunload→dismiss()→user stays on page

⬡ One-time vs Persistent Handler

Use page.on() for dialogs that appear multiple times in a test. Use page.waitForEvent('dialog') for a one-time dialog you want to handle exactly once without a persistent listener.

page.on('dialog', handler)→fires every time a dialog appears
page.waitForEvent('dialog')→resolves once, returns Dialog object
Remove persistent listener:
page.removeListener('dialog', handler)
Best Practices — Session 4
  • ✓
    Always register dialog handlers before the action that triggers them. The 'dialog' event fires synchronously in the browser, so a late-registered handler will never catch it.
  • ✓
    Always assert dialog.message() inside the handler before accepting/dismissing — this verifies the dialog shows the correct text, not just that one appeared.
  • ✓
    Use dialog.type() to write a single handler that routes different dialog types: alert → accept(), confirm → check expected flow, prompt → accept with test data.
  • ✓
    For one-time dialogs, prefer page.waitForEvent('dialog') over page.on() — it's cleaner, doesn't need cleanup with removeListener(), and makes the test flow more readable.
  • ✓
    If your test has multiple dialogs in sequence, maintain a response queue array in the handler and shift() one response per dialog call — this handles multi-step dialog flows cleanly.
  • ✓
    Always await dialog.accept() and dialog.dismiss() — they return Promises even though the dialog appears synchronous. Not awaiting causes test instability.
  • ✓
    For beforeunload testing, note that calling page.goto() from a page with unsaved changes may or may not trigger it depending on browser behavior — use page.close() with a registered handler for reliable beforeunload testing.
Interview Prep — 5 SDET Q&A
Q1: What happens if a dialog appears and no handler is registered in Playwright? ▶
Playwright auto-dismisses dialogs if no 'dialog' event handler is registered. For alert — it's like clicking OK. For confirm — it's like clicking Cancel (returns false to JS). For prompt — it's like clicking Cancel (returns null). For beforeunload — it's like staying on the page. This default behavior means your tests won't hang unexpectedly on an unhandled dialog. However, it also means you won't be able to verify the dialog message, test confirm acceptance flows, or provide prompt input — for those scenarios you must register a handler explicitly.
Q2: How do you test a delete confirmation flow that uses window.confirm()? ▶
I set up the handler before clicking Delete: page.on('dialog', async (dialog) => { expect(dialog.type()).toBe('confirm'); expect(dialog.message()).toContain('Are you sure you want to delete'); await dialog.accept(); // clicks OK }). Then I click the delete button: await page.click('#delete-btn'). Then assert the item is gone: await expect(page.locator('.item-row')).toHaveCount(0). For the dismiss case (user clicks Cancel), I use dialog.dismiss() and verify the item still exists. Two separate test cases cover both paths.
Q3: How do you handle a window.prompt() dialog in Playwright? ▶
prompt() dialogs have an input field. I handle them like this: page.on('dialog', async (dialog) => { expect(dialog.type()).toBe('prompt'); expect(dialog.message()).toBe('Enter your name:'); expect(dialog.defaultValue()).toBe('Guest'); // check pre-filled value await dialog.accept('Alice'); // fills the input AND clicks OK }). Calling dialog.accept('Alice') fills the prompt's input field with 'Alice' and confirms — the calling JS code receives 'Alice' as the return value. Calling dialog.dismiss() instead makes JS receive null regardless of any input.
Q4: How do you handle multiple sequential dialogs in one test? ▶
I use a response queue pattern: const responses = ['accept', 'dismiss', 'accept']; page.on('dialog', async (dialog) => { const response = responses.shift(); if (response === 'accept') { await dialog.accept(); } else { await dialog.dismiss(); } }). Each dialog invocation pops one response from the queue. This is clean and deterministic. For prompts in the queue, I'd use objects: [{action:'accept', text:'Alice'}, {action:'dismiss'}]. Then in the handler: if (r.action === 'accept') await dialog.accept(r.text ?? ''). This scales to any number of dialogs in any order.
Q5: How do you test the beforeunload dialog — the "Leave site?" warning? ▶
beforeunload dialogs appear when a user tries to navigate away from a page with unsaved changes. Testing it: First, navigate to the form page and start filling it (triggering the unsaved state). Register handler: page.on('dialog', async (dialog) => { expect(dialog.type()).toBe('beforeunload'); await dialog.dismiss(); // dismiss = stay on page }). Then trigger navigation: await page.close() or await page.goBack(). If the dialog appears, the handler dismisses it and the page stays. To test the "leave" path: await dialog.accept() — this confirms navigation and the page unloads. Note: Some frameworks use custom confirm() dialogs instead of real beforeunload — those are handled identically to confirm() dialogs.

alert-handling.ts
// alert-handling.ts
// PURPOSE: Handle window.alert() dialogs in Playwright
// alert() has one button (OK) — it just notifies the user

import { test, expect } from '@playwright/test';

test('handle alert dialog', async ({ page }) => {

  await page.goto('https://myapp.com/notifications');

  // Track dialog message for assertion
  let dialogMessage = '';

  // STEP 1: Register handler BEFORE the action that triggers it
  page.on('dialog', async (dialog) => {
    // Capture the dialog message for assertion
    dialogMessage = dialog.message();
    console.log(`📢 Dialog type: ${dialog.type()}`);    // 'alert'
    console.log(`📢 Message: ${dialog.message()}`);         // 'Success!'

    // accept() = click OK on the alert
    await dialog.accept();
  });

  // STEP 2: Trigger the action that fires the alert
  await page.click('#save-settings-btn');

  // STEP 3: Assert the dialog showed the correct message
  expect(dialogMessage).toBe('Settings saved successfully!');

  // STEP 4: Assert page state after dialog is dismissed
  await expect(page.locator('.settings-status'))
    .toContainText('Saved');
});
confirm-dialog.ts
// confirm-dialog.ts
// PURPOSE: Handle window.confirm() — OK returns true, Cancel returns false
// USE CASE: Delete confirmations, dangerous action warnings

import { test, expect } from '@playwright/test';

test.describe('Delete confirmation dialog', () => {

  test('accept confirm → item deleted', async ({ page }) => {

    await page.goto('https://myapp.com/items');
    await expect(page.locator('.item-row')).toHaveCount(3);

    // Register handler: accept the confirm dialog
    page.on('dialog', async (dialog) => {
      expect(dialog.type()).toBe('confirm');
      expect(dialog.message()).toContain('Are you sure you want to delete');
      await dialog.accept(); // ← clicks OK → JS confirm() returns true
    });

    await page.click('.delete-btn');

    // Item was deleted — now 2 remain
    await expect(page.locator('.item-row')).toHaveCount(2);
  });

  test('dismiss confirm → item NOT deleted', async ({ page }) => {

    await page.goto('https://myapp.com/items');
    await expect(page.locator('.item-row')).toHaveCount(3);

    // Register handler: dismiss the confirm dialog (click Cancel)
    page.on('dialog', async (dialog) => {
      await dialog.dismiss(); // ← clicks Cancel → JS confirm() returns false
    });

    await page.click('.delete-btn');

    // Nothing deleted — still 3 items
    await expect(page.locator('.item-row')).toHaveCount(3);
  });
});

// prompt-dialog.ts
// PURPOSE: Handle window.prompt() — passes text input to dialog
// KEY: dialog.accept('text') fills input AND clicks OK in one call

import { test, expect } from '@playwright/test';

test('handle prompt dialog with user input', async ({ page }) => {

  await page.goto('https://myapp.com/rename');

  // Use waitForEvent for one-time dialogs — cleaner than page.on()
  const dialogPromise = page.waitForEvent('dialog');

  // Trigger the prompt (app calls: prompt('Enter new name:', 'Default Name'))
  await page.click('#rename-btn');

  // Await and capture the dialog
  const dialog = await dialogPromise;

  // Verify dialog properties
  expect(dialog.type()).toBe('prompt');
  expect(dialog.message()).toBe('Enter new name:');
  expect(dialog.defaultValue()).toBe('Default Name'); // pre-filled value

  // Accept with custom text → JS receives 'My New Project' as return value
  await dialog.accept('My New Project');

  // Page should update with the new name
  await expect(page.locator('.project-title'))
    .toHaveText('My New Project');
});

test('dismiss prompt → no rename happens', async ({ page }) => {

  await page.goto('https://myapp.com/rename');

  page.on('dialog', async (dialog) => {
    await dialog.dismiss(); // Cancel → JS prompt() returns null
  });

  await page.click('#rename-btn');

  // Title unchanged since user cancelled
  await expect(page.locator('.project-title'))
    .toHaveText('Default Name');
});

// beforeunload.ts
// PURPOSE: Handle the beforeunload "Leave site?" dialog
// TRIGGER: User navigates away from a page with unsaved form data

import { test, expect } from '@playwright/test';

test('beforeunload: dismiss keeps user on page', async ({ page }) => {

  await page.goto('https://myapp.com/edit-article');

  // Start editing (creates unsaved state → beforeunload will fire)
  await page.fill('textarea#content', 'My unsaved article content');

  // Register handler for beforeunload
  page.on('dialog', async (dialog) => {
    expect(dialog.type()).toBe('beforeunload');
    // dismiss() = "Stay on page" / "Cancel"
    await dialog.dismiss();
  });

  // Try to navigate away — triggers beforeunload
  await page.goBack();

  // Should still be on edit page (navigation was blocked)
  await expect(page).toHaveURL(/edit-article/);
  await expect(page.locator('textarea#content'))
    .toHaveValue('My unsaved article content');
});

test('beforeunload: accept lets user leave', async ({ page }) => {

  await page.goto('https://myapp.com/edit-article');
  await page.fill('textarea#content', 'Unsaved content');

  page.on('dialog', async (dialog) => {
    // accept() = "Leave page" / "OK"
    await dialog.accept();
  });

  // Navigate away — accepts the warning → user leaves
  await page.goBack();

  // Should be on previous page now
  await expect(page).toHaveURL(/\/articles/);
});
multi-dialog.ts
// multi-dialog.ts
// PURPOSE: Handle multiple sequential dialogs in one test
// PATTERN: Response queue — shifts one response per dialog

import { test, expect, Dialog } from '@playwright/test';

type DialogResponse = {
  action: 'accept' | 'dismiss';
  text?: string;           // only for prompt dialogs
  expectedMsg?: string;   // expected message to assert
};

test('multi-step flow with sequential dialogs', async ({ page }) => {

  await page.goto('https://myapp.com/batch-actions');

  // Define responses queue in the ORDER dialogs will appear
  const responses: DialogResponse[] = [
    { action: 'accept', expectedMsg: 'Archive 5 items?' },       // Dialog 1
    { action: 'accept', text: 'archive-2024', expectedMsg: 'Archive name:' }, // Dialog 2 (prompt)
    { action: 'dismiss', expectedMsg: 'Also delete originals?' }, // Dialog 3
  ];

  // Single handler manages all dialogs using the queue
  page.on('dialog', async (dialog: Dialog) => {
    const response = responses.shift(); // pop first response
    if (!response) throw new Error('Unexpected dialog!');

    if (response.expectedMsg) {
      expect(dialog.message()).toContain(response.expectedMsg);
    }

    if (response.action === 'accept') {
      await dialog.accept(response.text ?? '');
    } else {
      await dialog.dismiss();
    }
  });

  // Trigger the action that fires all 3 dialogs in sequence
  await page.click('#batch-archive-btn');

  // All dialogs handled — verify final state
  await expect(page.locator('.archive-status'))
    .toContainText('Archived as: archive-2024');
  expect(responses.length).toBe(0); // all responses were consumed
});
dialog-types-router.ts
// dialog-types-router.ts
// PURPOSE: Single smart handler that routes all dialog types
// USE CASE: Reusable utility for test frameworks

import { test, expect, Page, Dialog } from '@playwright/test';

// ── Reusable dialog handler factory ─────────────────────────────
function createDialogHandler(options: {
  alert?: 'accept' | 'dismiss';
  confirm?: 'accept' | 'dismiss';
  prompt?: { action: 'accept' | 'dismiss'; text?: string };
  beforeunload?: 'accept' | 'dismiss';
  onMessage?: (msg: string, type: string) => void;
}) {
  return async (dialog: Dialog) => {
    const type = dialog.type();
    const msg  = dialog.message();

    options.onMessage?.(msg, type);
    console.log(`[Dialog] type=${type} message="${msg}"`);

    switch (type) {
      case 'alert':
        await (options.alert === 'dismiss'
          ? dialog.dismiss() : dialog.accept());
        break;
      case 'confirm':
        await (options.confirm === 'dismiss'
          ? dialog.dismiss() : dialog.accept());
        break;
      case 'prompt':
        const p = options.prompt;
        if (p?.action === 'dismiss') { await dialog.dismiss(); }
        else { await dialog.accept(p?.text ?? ''); }
        break;
      case 'beforeunload':
        await (options.beforeunload === 'accept'
          ? dialog.accept() : dialog.dismiss());
        break;
    }
  };
}

// ── Usage in test ────────────────────────────────────────────────
test('smart dialog router handles all types', async ({ page }) => {

  await page.goto('https://myapp.com');

  const messages: string[] = [];

  page.on('dialog', createDialogHandler({
    alert:       'accept',
    confirm:     'accept',
    prompt:      { action: 'accept', text: 'Test User' },
    beforeunload: 'dismiss',
    onMessage: (msg) => messages.push(msg),
  }));

  await page.click('#trigger-all-dialogs');
  console.log(`All dialog messages: ${messages.join(' | ')}`);
});
